## Cross-Phase Integration Findings

### INT-01 — CRITICAL: Payment Failure Token Blacklist Bug

**File:** `backend/src/wxcode_adm/billing/service.py:874-878`
**Path:** Phase 4 (Billing) → Phase 2 (Auth) + Phase 7 (Sessions)

`_handle_payment_failed` blacklists Redis keys using `token.token` (the opaque refresh token string) instead of `UserSession.access_token_jti` (the UUID embedded in the access token). The `is_token_blacklisted` check in `auth/dependencies.py` looks up `auth:blacklist:jti:{jti}` — a key named `auth:blacklist:jti:{refresh_token_string}` will never match.

**Impact:** Members of a payment-failed tenant retain valid access tokens until natural expiry. Refresh tokens are correctly deleted (blocking token renewal), and `require_active_subscription` correctly blocks tenant-scoped endpoints. But platform-level endpoints remain accessible.

**Fix:** Query `UserSession.access_token_jti` for each user and call `blacklist_jti(redis, jti)` — same pattern used correctly in `admin/service.py:suspend_tenant` (lines 447-454).

**Affected requirements:** BILL-03, BILL-05

### INT-02 — MEDIUM: Billing Admin Routes Bypass Admin JWT Audience Isolation

**File:** `backend/src/wxcode_adm/billing/router.py:46-60`
**Path:** Phase 4 (Billing Admin) → Phase 8 (Admin JWT Isolation)

Billing admin routes use a local `require_superuser` dependency (regular JWT + `is_superuser` check) instead of `require_admin` from `admin/dependencies.py` (which enforces `aud="wxcode-adm-admin"`). A regular user token where the user is a superuser can access billing admin endpoints, bypassing the admin audience isolation.

**Fix:** Replace `require_superuser` with `require_admin` from `admin/dependencies.py`.

**Affected requirements:** BILL-01

### INT-03 — LOW: Soft-Delete Tenant No Session Invalidation (By Design)

**File:** `backend/src/wxcode_adm/admin/service.py:537-594`

`soft_delete_tenant` does not blacklist JTIs or delete refresh tokens (by design per docstring). Members retain access to platform-level endpoints until token expiry. Tenant-scoped endpoints are correctly blocked by `get_tenant_context`.

**Affected requirements:** SADM-02

### Previous Audit Gap — CLOSED

The MFA→wxcode redirect gap (identified in the 2026-02-26 audit) was fixed by Phase 9 (commit `81ad93d`). `mfa_verify` now calls `get_redirect_url` + `create_wxcode_code` after `_issue_tokens`. Integration test at `test_oauth_mfa.py:788` verifies the complete flow including single-use code enforcement.
