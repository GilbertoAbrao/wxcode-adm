    1. **auth/service.py** — Add OAuth service functions at the end of the file:

       **get_github_email(client, token)** — Async function that:
       - Calls `client.get("user", token=token)` to get the profile
       - Extracts `profile["email"]`; if None, calls `client.get("user/emails", token=token)` to get the emails list
       - Selects the entry where `primary=True` from the emails list
       - Returns `(email: str, provider_user_id: str(profile["id"]))` tuple
       - Raises `OAuthEmailUnavailableError()` if no email found

       **get_google_userinfo(token)** — Simple extraction function:
       - Extracts from `token.get("userinfo", {})` or `token`
       - Returns `(email: str, provider_user_id: str(sub))` tuple
       - Must check `email_verified` claim from Google — if False, raise OAuthEmailUnavailableError
       - Raises `OAuthEmailUnavailableError()` if no email found

       **resolve_oauth_account(db, redis, provider, email, provider_user_id)** — Async function that implements the account resolution state machine:
       1. Look up existing OAuthAccount by `(provider, provider_user_id)`. If found, load the User and issue tokens (call existing token issuance: create_access_token + RefreshToken creation — extract this pattern from login() into a helper `_issue_tokens(db, redis, user)`).
       2. If no OAuthAccount found, look up User by email.
          - If User exists AND has password_hash (not None): return an OAuthLinkResponse with a `link_token` stored in Redis at `auth:oauth_link:{token}` with 300s TTL containing `{user_id, provider, provider_user_id}`. Per locked decision: prompt password to confirm ownership.
          - If User exists AND has no password_hash (OAuth-only user from another provider): Per locked decision "one OAuth provider per account", raise OAuthProviderAlreadyLinkedError. The user must unlink the existing provider first.
       3. If neither found: create new User (email_verified=False, password_hash=None, mfa_enabled=False) + OAuthAccount. Per locked decision: "Still require email OTP verification" — generate OTP code and enqueue verification email, same as signup flow. Return tokens with `is_new_user=True`.

       **confirm_oauth_link(db, redis, link_token, password)** — Async function:
       - Retrieve `auth:oauth_link:{link_token}` from Redis
       - If not found, raise InvalidTokenError
       - Extract user_id, provider, provider_user_id from Redis value (JSON)
       - Load User, verify password against password_hash
       - If password wrong, raise InvalidCredentialsError
       - Create OAuthAccount linking user to provider
       - Delete the Redis key
       - Issue tokens via `_issue_tokens`

       **_issue_tokens(db, redis, user)** — Helper extracted from login():
       - Revoke all existing refresh tokens for user (single-session)
       - Write shadow keys for old tokens
       - Create new access token + refresh token
       - Return TokenResponse
       - This avoids duplicating the token issuance logic across login, OAuth, and MFA verify

    2. **auth/router.py** — Add OAuth routes to `auth_api_router`:

       **GET /auth/oauth/{provider}/login** — OAuth redirect:
       - Validate `provider` is "google" or "github" — raise 400 if unknown
       - Get the OAuth client via `oauth.create_client(provider)`
       - Compute `redirect_uri = str(request.url_for("oauth_callback", provider=provider))`
       - Return `await client.authorize_redirect(request, redirect_uri)`
       - Apply `@limiter.limit(settings.RATE_LIMIT_AUTH)` rate limiting
       - Name the route function `oauth_callback` (used by url_for)

       **GET /auth/oauth/{provider}/callback** — OAuth callback:
       - Get OAuth client, call `token = await client.authorize_access_token(request)`
       - Extract userinfo based on provider:
         - "google": call `get_google_userinfo(token)`
         - "github": call `get_github_email(client, token)`
       - Call `resolve_oauth_account(db, redis, provider, email, provider_user_id)`
       - The resolve function returns either OAuthCallbackResponse or OAuthLinkResponse
       - Return the response directly
       - Rate limit with `@limiter.limit(settings.RATE_LIMIT_AUTH)`

       **POST /auth/oauth/link/confirm** — Confirm account link with password:
       - Accept `OAuthLinkConfirmRequest` body
       - Call `confirm_oauth_link(db, redis, body.link_token, body.password)`
       - Return `OAuthCallbackResponse` with tokens
       - Rate limit with `@limiter.limit(settings.RATE_LIMIT_AUTH)`

       IMPORTANT: For the OAuth callback route, use `name="oauth_callback"` in the decorator to match the `request.url_for("oauth_callback", provider=provider)` call in the redirect route.

       IMPORTANT: Both OAuth routes require `request: Request` as the first parameter for both rate limiting (slowapi) and authlib session access.

       Add audit log entries for: oauth_login (in callback on success), oauth_link (in link confirm on success).
