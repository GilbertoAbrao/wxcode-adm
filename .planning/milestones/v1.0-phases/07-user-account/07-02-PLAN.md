1. Ensure `backend/src/wxcode_adm/users/__init__.py` exists (may already be empty from Phase 1 scaffolding).

2. Create `backend/src/wxcode_adm/users/schemas.py`:
   ```python
   class UserProfileResponse(BaseModel):
       """GET /users/me response."""
       id: str
       email: str
       email_verified: bool
       display_name: str | None
       avatar_url: str | None
       mfa_enabled: bool

       model_config = ConfigDict(from_attributes=True)

   class UpdateProfileRequest(BaseModel):
       """PATCH /users/me request — all fields optional (partial update)."""
       display_name: str | None = Field(None, min_length=1, max_length=255)
       email: EmailStr | None = None

   class UpdateProfileResponse(BaseModel):
       """PATCH /users/me response."""
       message: str
       profile: UserProfileResponse

   class AvatarUploadResponse(BaseModel):
       """POST /users/me/avatar response."""
       avatar_url: str
       message: str

   class ChangePasswordRequest(BaseModel):
       """POST /users/me/change-password request."""
       current_password: str
       new_password: str = Field(min_length=8, max_length=128)

   class ChangePasswordResponse(BaseModel):
       """POST /users/me/change-password response."""
       message: str
   ```

3. Create `backend/src/wxcode_adm/users/service.py`:

   **get_profile(user: User) -> dict:**
   Return dict with id (str), email, email_verified, display_name, avatar_url, mfa_enabled. Straightforward — User model has all fields after Plan 01.

   **update_profile(db, redis, user, data: UpdateProfileRequest) -> User:**
   - If `data.display_name` is not None, set `user.display_name = data.display_name`.
   - If `data.email` is not None AND `data.email != user.email`:
     - Check for email uniqueness (query User by new email, raise ConflictError if taken).
     - Set `user.email = data.email`.
     - Set `user.email_verified = False`.
     - Generate and send new OTP code (reuse auth service pattern: generate 6-digit code, store in Redis `auth:otp:{user.id}`, enqueue verification email via arq).
   - Return updated user.

   **upload_avatar(user: User, file: UploadFile, db: AsyncSession) -> str:**
   - Validate content type: must be `image/jpeg` or `image/png`. Raise AppError(400) otherwise.
   - Read file content. Validate size <= 2MB (2 * 1024 * 1024 bytes). Raise AppError(400) if too large.
   - Use Pillow to open, resize to 256x256 (Image.LANCZOS resampling), convert to RGB, save as JPEG.
   - Save to `{AVATAR_UPLOAD_DIR}/{user.id}.jpg`. Create directory if not exists (`os.makedirs(exist_ok=True)`).
   - Set `user.avatar_url = f"/avatars/{user.id}.jpg"` (relative path only per research recommendation).
   - Return the avatar_url string.

   **change_password(db, redis, user, current_password: str, new_password: str) -> None:**
   - If `user.password_hash` is None (OAuth-only user with no password):
     Raise AppError(400, "OAUTH_ONLY_ACCOUNT", "Cannot change password — account uses OAuth only. Use password reset to set a password.").
   - Verify current_password against user.password_hash using `pwdlib` hasher (same pattern as auth.service.login). Raise InvalidCredentialsError if wrong.
   - Hash new_password and set `user.password_hash = hash_password(new_password)`.
   - Invalidate ALL OTHER sessions for this user:
     - Query all RefreshToken rows for user.id.
     - For each, look up the associated UserSession to get access_token_jti.
     - Blacklist each access_token_jti in Redis (reuse `blacklist_access_token` from auth.service).
     - Delete all RefreshToken rows for user.id (CASCADE deletes UserSession rows).
     NOTE: "all OTHER sessions" — to keep the current session alive, the caller (router) must pass the current JTI so this function can skip it. Add `current_jti: str | None = None` parameter. If current_jti is provided, skip the RefreshToken whose UserSession.access_token_jti matches current_jti.
   - Import hash_password from auth.service (or from pwdlib directly — follow existing pattern in auth/service.py).
  </action>

   ```python
   users_router = APIRouter(prefix="/users", tags=["users"])
   ```

   **GET /users/me** — Replace the existing `/auth/me` endpoint:
   - Depends on `require_verified`.
   - Calls `service.get_profile(user)`.
   - Returns `UserProfileResponse`.

   **PATCH /users/me** — Partial profile update:
   - Depends on `require_verified`, `get_session`, `get_redis`.
   - Accepts `UpdateProfileRequest` body.
   - If body has no fields set (both None), return 400 "No fields to update".
   - Calls `service.update_profile(db, redis, user, body)`.
   - Returns `UpdateProfileResponse` with message and updated profile.
   - Write audit: action="profile_update", resource_type="user", actor_id=user.id.

   **POST /users/me/avatar** — Avatar upload:
   - Depends on `require_verified`, `get_session`.
   - Accepts `file: UploadFile`.
   - Calls `service.upload_avatar(user, file, db)`.
   - Returns `AvatarUploadResponse`.
   - Write audit: action="avatar_upload", resource_type="user", actor_id=user.id.

   **POST /users/me/change-password** — Password change:
   - Depends on `require_verified`, `get_session`, `get_redis`.
   - Depends on `get_current_jti` (from auth.dependencies) to preserve current session.
   - Accepts `ChangePasswordRequest` body.
   - Calls `service.change_password(db, redis, user, body.current_password, body.new_password, current_jti=jti)`.
   - Returns `ChangePasswordResponse(message="Password changed successfully")`.
   - Write audit: action="password_change", resource_type="user", actor_id=user.id.
   - All password endpoints use `@limiter.limit(settings.RATE_LIMIT_AUTH)` for brute-force protection.
   - All endpoint functions accept `request: Request` as first parameter (slowapi requirement per [05-01] decision).
