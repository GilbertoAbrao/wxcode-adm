       - `TenantListItem(BaseModel)`: `id: UUID`, `name: str`, `slug: str`, `is_suspended: bool`, `is_deleted: bool`, `plan_name: str | None`, `plan_slug: str | None`, `member_count: int`, `created_at: datetime`. Use `model_config = ConfigDict(from_attributes=True)`.

Use `model_config = ConfigDict(from_attributes=True)`.

    2. Update `backend/src/wxcode_adm/admin/service.py` — add tenant management functions:

       a. `list_tenants(db, limit=20, offset=0, plan_slug=None, status=None) -> tuple[list[dict], int]`:
          - Build base query: `select(Tenant)` with optional filters.
          - If `status == "suspended"`: filter `Tenant.is_suspended == True`.
          - If `status == "deleted"`: filter `Tenant.is_deleted == True`.
          - If `status == "active"`: filter `Tenant.is_suspended == False, Tenant.is_deleted == False`.
          - If `plan_slug`: join TenantSubscription -> Plan, filter `Plan.slug == plan_slug`.
          - Count query: `select(func.count()).select_from(Tenant)` with same filters.
          - For each tenant, compute member_count via a correlated subquery: `select(func.count(TenantMembership.id)).where(TenantMembership.tenant_id == Tenant.id).correlate(Tenant).scalar_subquery()`.
          - Main query selects `(Tenant, member_count_subq.label("member_count"))`.
          - For plan info, use `selectinload(Tenant.subscription)` if Tenant has subscription relationship, OR do a separate query joining TenantSubscription + Plan for each tenant row. The simpler approach: after fetching tenants, batch-load subscriptions. Actually, use `outerjoin(TenantSubscription, TenantSubscription.tenant_id == Tenant.id).outerjoin(Plan, Plan.id == TenantSubscription.plan_id)` and add `Plan.name.label("plan_name"), Plan.slug.label("plan_slug")` to select.
          - Order by `Tenant.created_at.desc()`, apply `limit` and `offset`.
          - Return `(items_as_dicts, total)`.

       b. `get_tenant_detail(db, tenant_id) -> dict`:
          - Load Tenant by ID. If not found, raise `NotFoundError(error_code="TENANT_NOT_FOUND")`.
          - Load TenantSubscription with eager Plan for this tenant.
          - Count members.
          - Return dict with all TenantDetailResponse fields.

       c. `suspend_tenant(db, redis, tenant_id, reason, actor_id) -> Tenant`:
          - Load Tenant. Raise NotFoundError if not found.
          - Raise ConflictError(error_code="ALREADY_SUSPENDED") if already suspended.
          - Set `tenant.is_suspended = True`.
          - Get all user_ids via TenantMembership for this tenant.
          - Blacklist active access tokens: query UserSession.access_token_jti for those user_ids, call `blacklist_jti(redis, jti)` for each.
          - Delete all RefreshToken rows for those user_ids: `delete(RefreshToken).where(RefreshToken.user_id.in_(user_ids))`.
          - write_audit: action="suspend_tenant", resource_type="tenant", resource_id=str(tenant_id), details={"reason": reason}.
          - Return tenant.

       d. `reactivate_tenant(db, tenant_id, reason, actor_id) -> Tenant`:
          - Load Tenant. Raise NotFoundError if not found.
          - Raise ConflictError(error_code="NOT_SUSPENDED") if not suspended.
          - Set `tenant.is_suspended = False`.
          - write_audit: action="reactivate_tenant", resource_type="tenant", resource_id=str(tenant_id), details={"reason": reason}.
          - Return tenant.

       e. `soft_delete_tenant(db, tenant_id, reason, actor_id) -> Tenant`:
          - Load Tenant. Raise NotFoundError if not found.
          - Raise ConflictError(error_code="ALREADY_DELETED") if already soft-deleted.
          - Set `tenant.is_deleted = True`.
          - write_audit: action="soft_delete_tenant", resource_type="tenant", resource_id=str(tenant_id), details={"reason": reason}.
          - Return tenant. Note: No session invalidation needed — the `get_tenant_context` enforcement hook (from Plan 01) handles blocking deleted tenants on next request.

    Import from existing modules: `blacklist_jti` from `wxcode_adm.auth.service`, `write_audit` from `wxcode_adm.audit.service`, `RefreshToken` and `UserSession` from `wxcode_adm.auth.models`, `TenantMembership` and `Tenant` from `wxcode_adm.tenants.models`, `TenantSubscription` and `Plan` from `wxcode_adm.billing.models`, `NotFoundError` and `ConflictError` from `wxcode_adm.common.exceptions`.
