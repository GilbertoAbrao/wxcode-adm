## Tech Debt

### Phase 12: Design System Foundation

1. **Sidebar /settings dead link** — `Sidebar.tsx` line 76 includes a `<Link href="/settings">` with Settings gear icon, but no `(app)/settings/page.tsx` exists. Clicking it returns a 404. Settings page was never in scope for any v2.0 phase.

2. **Static main dashboard** — `(app)/page.tsx` shows hardcoded "—" values and the text "Phases 13–17 will populate this dashboard with live data." No subsequent phase added live data hooks. This is the primary landing page after login. No v2.0 requirement specifies a live tenant dashboard.

### Phase 17: Super-Admin UI

3. **Admin post-login redirect** — `AdminAuthProvider` and `admin/login/page.tsx` both redirect to `/admin/tenants` after login. After Phase 18 added `/admin/dashboard` as the logical admin home page, the redirect should arguably target `/admin/dashboard` instead.

4. **Import pattern inconsistency** — `admin/users/page.tsx` imports design system components from individual paths (`@/components/ui/GlowButton`) instead of the barrel export (`@/components/ui`). Functionally identical but inconsistent with the project pattern.

### Total: 4 items across 2 phases
