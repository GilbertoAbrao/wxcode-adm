Create `frontend/src/hooks/useUserAccount.ts` as a "use client" file with TanStack Query hooks for user account management. Model after useAuth.ts patterns (useMutation, useQuery, apiClient, typed request/response interfaces).

Define these TypeScript interfaces (matching backend Phase 7 schemas):

```typescript
export interface UserProfile {
  id: string;
  email: string;
  display_name: string | null;
  avatar_url: string | null;
  email_verified: boolean;
  mfa_enabled: boolean;
}

export interface UpdateProfileRequest {
  display_name?: string;
  email?: string;
}

export interface UpdateProfileResponse {
  id: string;
  email: string;
  display_name: string | null;
  avatar_url: string | null;
}

export interface UploadAvatarResponse {
  avatar_url: string;
}

export interface ChangePasswordRequest {
  current_password: string;
  new_password: string;
}

export interface ChangePasswordResponse {
  message: string;
}

export interface UserSession {
  id: string;
  device_info: string | null;
  ip_address: string | null;
  last_active_at: string;
  created_at: string;
  is_current: boolean;
}

export interface SessionListResponse {
  sessions: UserSession[];
}
```

Implement these hooks:

1. `useCurrentUser()` — useQuery with queryKey `["user", "me"]`, queryFn calls `apiClient<UserProfile>("/users/me")`. staleTime: 30_000 (30 seconds, avoids excessive refetches while user edits).

2. `useUpdateProfile()` — useMutation, mutationFn calls `apiClient<UpdateProfileResponse>("/users/me", { method: "PATCH", body: JSON.stringify(data) })`. onSuccess: invalidates `["user", "me"]` query key via queryClient.

3. `useUploadAvatar()` — useMutation, mutationFn accepts a `File` object. Builds FormData with `formData.append("file", file)` then calls:
   ```typescript
   apiClient<UploadAvatarResponse>("/users/me/avatar", {
     method: "POST",
     body: formData,
     headers: {},  // Let browser set Content-Type with boundary for multipart
   })
   ```
   IMPORTANT: Do NOT set Content-Type header manually for multipart — pass `headers: {}` to override the default "application/json" that apiClient sets. The apiClient merges caller headers over defaults, so passing an empty headers object lets apiClient set Authorization only. Actually, look at apiClient source: it builds headers object starting with `{ "Content-Type": "application/json" }` then merges caller headers. To send multipart without Content-Type, override by passing `headers: { "Content-Type": undefined }` — but that won't work in Record<string,string>. Instead, create a modified fetch call directly bypassing apiClient's Content-Type, OR restructure the call.

   Correct approach: do NOT use apiClient for avatar upload. Make a direct fetch call:
   ```typescript
   import { getAccessToken } from "@/lib/auth";
   import { API_BASE } from "@/lib/api-client";
   // in mutationFn:
   const token = getAccessToken();
   const formData = new FormData();
   formData.append("file", file);
   const response = await fetch(`${API_BASE}/api/v1/users/me/avatar`, {
     method: "POST",
     headers: token ? { Authorization: `Bearer ${token}` } : {},
     body: formData,
   });
   if (!response.ok) {
     const body = await response.json().catch(() => ({}));
     throw new Error(body?.detail || "Avatar upload failed");
   }
   return response.json() as Promise<UploadAvatarResponse>;
   ```
   onSuccess: invalidates `["user", "me"]`.

4. `useChangePassword()` — useMutation, calls `apiClient<ChangePasswordResponse>("/users/me/change-password", { method: "POST", body: JSON.stringify(data) })`.

5. `useUserSessions()` — useQuery with queryKey `["user", "sessions"]`, queryFn calls `apiClient<SessionListResponse>("/users/me/sessions")`. staleTime: 10_000.

6. `useRevokeSession()` — useMutation, mutationFn accepts `sessionId: string`, calls `apiClient<void>("/users/me/sessions/${sessionId}", { method: "DELETE" })`. onSuccess: invalidates `["user", "sessions"]`.

7. `useRevokeAllSessions()` — useMutation, calls `apiClient<void>("/users/me/sessions", { method: "DELETE" })`. onSuccess: invalidates `["user", "sessions"]`.

All query hooks use `useQueryClient()` for invalidation. Import `useQuery`, `useMutation`, `useQueryClient` from `@tanstack/react-query`.
