  <name>Task 1: Create useAdminTenants hooks for tenant list and moderation actions</name>
  <files>frontend/src/hooks/useAdminTenants.ts</files>
  <action>
    Create TanStack Query hooks for admin tenant management endpoints, following the same pattern as useBilling.ts and useTenant.ts.

    **TypeScript interfaces** (matching backend admin/schemas.py exactly):
    - `TenantListItem`: { id: string, name: string, slug: string, is_suspended: boolean, is_deleted: boolean, plan_name: string | null, plan_slug: string | null, member_count: number, created_at: string }
    - `TenantListResponse`: { items: TenantListItem[], total: number }
    - `AdminActionRequest`: { reason: string }

    **Query key constants:**
    ```
    ADMIN_TENANT_KEYS = {
      list: (params) => ["admin", "tenants", params] as const,
    }
    ```

    **Hooks:**
    - `useAdminTenants(params: { limit?: number, offset?: number, plan_slug?: string | null, status?: string | null })` — useQuery calling GET /admin/tenants with query params. Build URL with URLSearchParams, skip null/undefined params. staleTime: 30s. Returns TenantListResponse. Query key includes all params for proper caching per filter combination.
    - `useSuspendTenant()` — useMutation calling POST /admin/tenants/{tenant_id}/suspend with { reason } body. On success: invalidate all admin tenant queries via `queryClient.invalidateQueries({ queryKey: ["admin", "tenants"] })`. Variables type: `{ tenant_id: string, reason: string }`.
    - `useReactivateTenant()` — useMutation calling POST /admin/tenants/{tenant_id}/reactivate with { reason } body. Same invalidation pattern. Same variables type.

    All hooks use `adminApiClient` from `@/lib/admin-api-client` (NOT apiClient).
    All hooks are "use client" directive at top.
  </action>
  <verify>
    Run `cd frontend && npx tsc --noEmit` — zero TypeScript errors. Verify useAdminTenants.ts exports the 3 hooks and 2 interfaces.
  </verify>

  <files>frontend/src/app/admin/tenants/page.tsx</files>
  <action>
    Create the /admin/tenants page following the established page patterns (team/page.tsx, billing/page.tsx) but adapted for admin tenant management.

    **Page structure:**
    - "use client" directive
    - Page header: "Tenant Management" title with a brief description
    - Admin navigation bar at top: simple horizontal nav with links to /admin/tenants (active), /admin/users. Use cyan-400 active state matching wxCode brand. Include a "Logout" button on the right that calls admin logout and redirects to /admin/login.
    - Filter bar: two dropdown selects (plan filter + status filter) rendered as HTML <select> elements styled with dark theme (bg-zinc-900, border-zinc-700, text-zinc-300). Options:
      - Plan: "All Plans" (value=""), then list common plan slugs (or leave as free text since plans are dynamic — use a text input or just "All Plans" option with the most common: "free", "starter", "pro", "enterprise")
      - Status: "All" (value=""), "Active", "Suspended", "Deleted"
    - IMPORTANT: Since plan slugs are dynamic, use a simple text/select approach. Provide "All Plans" as default and let admin type a plan slug in a GlowInput, or use a simple <select> with "All Plans" + common values. Best approach: use a small GlowInput for plan slug filter (placeholder "Filter by plan slug...") and a <select> for status filter.

    **Tenant table:**
    - Rendered as a responsive table (overflow-x-auto on mobile)
    - Columns: Name, Slug, Plan, Status, Members, Created, Actions
    - Name column: tenant name as primary text
    - Slug column: monospace text (font-mono text-zinc-500)
    - Plan column: plan_name or "No plan" in zinc-500
    - Status column: badge with color coding:
      - Active (not suspended, not deleted): green badge (bg-emerald-400/10 text-emerald-400)
      - Suspended: amber badge (bg-amber-400/10 text-amber-400)
      - Deleted: red badge (bg-rose-400/10 text-rose-400)
    - Members column: member_count number
    - Created column: formatted date (toLocaleDateString)
    - Actions column: contextual buttons
      - If active: "Suspend" button (amber text, hover bg)
      - If suspended: "Reactivate" button (emerald text, hover bg)
      - If deleted: no actions (read-only)

    **Suspend/Reactivate flow:**
    - When admin clicks "Suspend" or "Reactivate", show an inline reason input row below the tenant row (NOT a modal — follow the same inline confirmation pattern from team/page.tsx with confirmRemove)
    - State: `actionTenant: { id: string, action: "suspend" | "reactivate" } | null`
    - When actionTenant is set, render a row with a GlowInput for reason + "Confirm" GlowButton + "Cancel" text button
    - On confirm: call useSuspendTenant or useReactivateTenant with { tenant_id, reason }
    - On success: clear actionTenant state (mutation invalidation refreshes the list)
    - On error: show error message inline (ApiError.message)

    **Pagination:**
    - State: `page` (number, starting at 0), `limit` = 20
    - offset = page * limit
    - Show "Showing X-Y of Z tenants" text below the table
    - Previous/Next buttons (disabled at boundaries)
    - Use GlowButton variant="ghost" for pagination buttons

    **Loading/Error/Empty states:**
    - Loading: LoadingSkeleton
    - Error: ErrorState with retry
    - Empty: EmptyState with "No tenants found" message

    **Table styling (dark theme):**
    - Table: w-full border-collapse
    - Header: bg-zinc-900/50, text-zinc-400, text-sm, uppercase tracking-wider
    - Rows: border-b border-zinc-800, hover:bg-zinc-900/30
    - Cells: px-4 py-3 text-sm text-zinc-300
