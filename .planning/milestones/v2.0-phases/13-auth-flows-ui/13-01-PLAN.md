1. Install react-hook-form, @hookform/resolvers, and zod:
   `cd frontend && pnpm add react-hook-form @hookform/resolvers zod`

2. Create `frontend/src/lib/api-client.ts` — a typed fetch wrapper:
   - Export `API_BASE` constant: `process.env.NEXT_PUBLIC_API_URL || "http://localhost:8040"` (used server-side and in api-client)
   - Export async function `apiClient<T>(endpoint: string, options?: RequestInit & { skipAuth?: boolean }): Promise<T>`
   - Prepend `${API_BASE}/api/v1` to the endpoint
   - Set Content-Type: application/json by default
   - If `skipAuth` is not true, read access token from `getAccessToken()` (imported from auth.ts) and add `Authorization: Bearer ${token}` header
   - On 401 response: attempt silent refresh via `refreshTokens()` from auth.ts. If refresh succeeds, retry the original request once with the new token. If refresh fails, call `clearTokens()` and throw.
   - On non-ok responses: parse error body `{ detail?: string, error_code?: string, message?: string }` and throw an `ApiError` class with `status`, `message`, `errorCode` fields
   - Export `class ApiError extends Error { status: number; errorCode: string; }`

3. Create `frontend/src/lib/auth.ts` — in-memory token storage:
   - Token strategy: store access_token and refresh_token in module-scoped variables (NOT localStorage — XSS-safe). Tokens survive within a tab session but are lost on full page reload (user re-logs in). This is the simplest secure approach for an SPA that redirects to wxcode after login anyway.
   - Export `getAccessToken(): string | null`
   - Export `getRefreshToken(): string | null`
   - Export `setTokens(access: string, refresh: string): void`
   - Export `clearTokens(): void`
   - Export async `refreshTokens(): Promise<boolean>` — calls POST /api/v1/auth/refresh with current refresh_token, updates stored tokens on success, returns true/false
   - Export `isAuthenticated(): boolean` — returns true if access_token is not null
  </action>

1. Create `frontend/src/providers/auth-provider.tsx`:
   - "use client" directive
   - Define `AuthUser` type: `{ id: string; email: string; display_name: string | null; avatar_url: string | null; email_verified: boolean; mfa_enabled: boolean }`
   - Define `AuthContextValue`: `{ user: AuthUser | null; isAuthenticated: boolean; isLoading: boolean; login: (tokens: { access_token: string; refresh_token: string }) => void; logout: () => Promise<void> }`
   - Create React context `AuthContext`
   - `AuthProvider` component:
     - On mount, if `isAuthenticated()` from auth.ts is true, fetch GET /api/v1/users/me via apiClient to populate user state
     - `login` function: calls `setTokens`, then fetches /users/me to populate user
     - `logout` function: calls POST /api/v1/auth/logout with refresh_token, then `clearTokens()`, then `router.push("/login")`
     - Export `useAuthContext()` hook that reads from AuthContext

2. Create `frontend/src/hooks/useAuth.ts`:
   - "use client" directive
   - Import useMutation from @tanstack/react-query, apiClient from lib/api-client, types matching backend schemas
   - Define TypeScript interfaces matching backend auth schemas:
     - `SignupRequest { email: string; password: string }`
     - `SignupResponse { message: string }`
     - `LoginRequest { email: string; password: string }`
     - `LoginResponse { access_token?: string; refresh_token?: string; mfa_required: boolean; mfa_token?: string; mfa_setup_required: boolean; wxcode_redirect_url?: string; wxcode_code?: string }`
     - `VerifyEmailRequest { email: string; code: string }`
     - `ForgotPasswordRequest { email: string }`
     - `ResetPasswordRequest { token: string; new_password: string }`
     - `MfaVerifyRequest { mfa_token: string; code: string; trust_device: boolean }`
     - `CreateWorkspaceRequest { name: string }`
   - Export `useSignup()` — useMutation calling POST /auth/signup with skipAuth: true
   - Export `useLogin()` — useMutation calling POST /auth/login with skipAuth: true
   - Export `useVerifyEmail()` — useMutation calling POST /auth/verify-email with skipAuth: true
   - Export `useResendVerification()` — useMutation calling POST /auth/resend-verification with skipAuth: true
   - Export `useForgotPassword()` — useMutation calling POST /auth/forgot-password with skipAuth: true
   - Export `useResetPassword()` — useMutation calling POST /auth/reset-password with skipAuth: true
   - Export `useMfaVerify()` — useMutation calling POST /auth/mfa/verify with skipAuth: true
   - Export `useCreateWorkspace()` — useMutation calling POST /onboarding/workspace (requires auth, so no skipAuth)
   - Export `useLogout()` — useMutation calling POST /auth/logout (requires auth)

4. Create `frontend/src/middleware.ts`:
   - Next.js edge middleware for route protection
   - Since tokens are in-memory (not cookies), middleware cannot read them server-side. Instead, use a lightweight approach:
     - Define `publicPaths` array: `/login`, `/signup`, `/verify-email`, `/forgot-password`, `/reset-password`, `/mfa-verify`, `/onboarding`
     - For public paths: allow through (NextResponse.next())
     - For protected paths: allow through (protection will be handled client-side by AuthProvider which checks isAuthenticated and redirects to /login if not)
   - NOTE: Since tokens are in-memory (not cookies), true server-side middleware protection is not possible. The middleware file is a placeholder that can be upgraded later if tokens move to httpOnly cookies. For now, client-side AuthProvider handles redirect logic.
