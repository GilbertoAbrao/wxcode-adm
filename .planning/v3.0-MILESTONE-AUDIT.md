## Tech Debt

### Non-Critical Items

1. **Plan limits not in wxcode-config response** — The wxcode engine cannot read `max_projects`, `max_output_projects`, `max_storage_gb`, `token_quota_5h`, `token_quota_weekly` from the config endpoint. Requires `TenantSubscription → Plan` join in `tenants/router.py:739-746`. This was outside v3.0 spec (Phase 22 deliverables listed only tenant-level fields) but will be needed when the wxcode engine implements limit enforcement. **Carry to next milestone.**

2. **decrypt_value not used in production** — `encrypt_value` is used for storage, but `decrypt_value` has no production caller. By design: the documented integration uses token exchange (`/auth/wxcode/exchange`), not direct token decryption. If a future engine-to-engine flow needs the raw Claude token, `decrypt_value` is ready.

3. **INTEGRATION-CONTRACT.md consistency** — The contract accurately documents what is implemented but will need updating when plan limits are added to wxcode-config.
