Wire slowapi rate limiting into the FastAPI application with a strict 5/minute limit on auth endpoints and a 60/minute global default on all other endpoints, backed by Redis for persistence across restarts.

Purpose: Protect login, signup, and password reset endpoints from brute-force attacks (PLAT-03), and apply a global rate limit to all authenticated endpoints to prevent API abuse.

Output: Rate limiting active on all endpoints; health and JWKS exempt; 429 responses with Retry-After header on limit breach.
</objective>

   - `RATE_LIMIT_AUTH: str = "5/minute"` — strict limit for auth endpoints
   - `RATE_LIMIT_GLOBAL: str = "60/minute"` — global default for all endpoints

2. **common/rate_limit.py** — Create the slowapi limiter singleton:
   ```python
   from slowapi import Limiter, _rate_limit_exceeded_handler
   from slowapi.util import get_remote_address
   from slowapi.errors import RateLimitExceeded
   from slowapi.middleware import SlowAPIASGIMiddleware
   from wxcode_adm.config import settings

   limiter = Limiter(
       key_func=get_remote_address,
       default_limits=[settings.RATE_LIMIT_GLOBAL],
       storage_uri=settings.REDIS_URL,
   )
   ```
   Re-export `_rate_limit_exceeded_handler`, `RateLimitExceeded`, and `SlowAPIASGIMiddleware` for convenient import in main.py.

3. **main.py** — In `create_app()`, AFTER the app is created but BEFORE routers are mounted:
   - Import from `common.rate_limit`: limiter, _rate_limit_exceeded_handler, RateLimitExceeded, SlowAPIASGIMiddleware
   - `app.state.limiter = limiter`
   - `app.add_exception_handler(RateLimitExceeded, _rate_limit_exceeded_handler)`
   - `app.add_middleware(SlowAPIASGIMiddleware)`

   Order matters: middleware must be added before routers so global default_limits apply.

CRITICAL RULES (from research):
- Use `SlowAPIASGIMiddleware` (NOT `SlowAPIMiddleware`) — the non-ASGI variant does not work correctly with async FastAPI.
- The `_rate_limit_exceeded_handler` automatically returns 429 with `Retry-After` header.
