<objective>
Session management endpoints, wxcode redirect with one-time code exchange, and wiring session metadata from HTTP requests into _issue_tokens.

Purpose: Users can see all their active sessions with rich metadata and revoke any session immediately (USER-03). After login, the wxcode redirect uses a one-time authorization code — token never appears in a URL (USER-04, locked decision). Session metadata (user_agent, ip_address) is extracted from the request at the router layer and passed through to _issue_tokens so UserSession rows have populated device/browser/IP/city fields (locked decision: rich session metadata).

Output: Session list/revoke endpoints on /users/me/sessions; wxcode code creation in login flow; POST /auth/wxcode/exchange endpoint for server-to-server code-for-token swap; all login/MFA/OAuth flows pass request metadata into _issue_tokens.
</objective>

   ```python
   class SessionResponse(BaseModel):
       """Individual session in the sessions list."""
       id: str              # UserSession.id (UUID as string)
       device_type: str | None
       browser_name: str | None
       browser_version: str | None
       ip_address: str | None
       city: str | None
       last_active_at: str | None  # ISO 8601 timestamp
       is_current: bool     # True if this session matches the requesting JTI
       created_at: str      # ISO 8601 timestamp

   class SessionListResponse(BaseModel):
       """GET /users/me/sessions response."""
       sessions: list[SessionResponse]

   class RevokeSessionResponse(BaseModel):
       """DELETE /users/me/sessions/{session_id} response."""
       message: str

   class RevokeAllSessionsResponse(BaseModel):
       """DELETE /users/me/sessions response."""
       message: str
       revoked_count: int
   ```

2. Add session service functions to `users/service.py`:

   **list_sessions(db, redis, user, current_jti: str) -> list[dict]:**
   - Query all UserSession rows where `user_id == user.id` joined with RefreshToken (to confirm session is still active — RefreshToken row exists).
   - For each session, check Redis key `auth:session:last_active:{session.access_token_jti}` — if exists, use that timestamp; otherwise fall back to `session.last_active_at` from DB.
   - Mark `is_current = (session.access_token_jti == current_jti)`.
   - Return list of dicts sorted by last_active_at descending (most recent first).

   **revoke_session(db, redis, user, session_id: str, current_jti: str) -> None:**
   - Load UserSession by `id == session_id AND user_id == user.id`. Raise NotFoundError if not found.
   - If `session.access_token_jti == current_jti`, raise AppError(400, "CANNOT_REVOKE_CURRENT", "Cannot revoke your current session. Use logout instead.") — locked decision: prevent accidental self-revocation.
   - Blacklist the access token JTI in Redis (immediate rejection per locked decision):
     `await blacklist_access_token(redis, access_token_jti)` — reuse existing helper. NOTE: blacklist_access_token currently takes a full JWT token string and decodes it. For session revocation, we have the JTI directly. Add a new helper `blacklist_jti(redis, jti)` that writes the Redis blacklist key directly: `auth:blacklist:jti:{jti}` with TTL = ACCESS_TOKEN_TTL_HOURS * 3600. OR modify existing code to also accept a raw jti. Prefer adding a small `blacklist_jti` function in auth/service.py for clarity.
   - Delete the RefreshToken row (CASCADE deletes UserSession row): `await db.execute(delete(RefreshToken).where(RefreshToken.id == session.refresh_token_id))`.

   **revoke_all_other_sessions(db, redis, user, current_jti: str) -> int:**
   - Query all UserSession rows for user.id WHERE `access_token_jti != current_jti`.
   - For each, blacklist the access_token_jti in Redis.
   - Delete all RefreshToken rows for user.id WHERE `id != (the current session's refresh_token_id)`. Get current session's refresh_token_id by querying UserSession WHERE access_token_jti == current_jti.
   - Return count of revoked sessions.

1. Add wxcode code helpers to `auth/service.py`:

   **create_wxcode_code(redis, user_id: str, access_token: str, refresh_token: str) -> str:**
   - Generate code: `code = secrets.token_urlsafe(32)`
   - Store in Redis: key = `auth:wxcode_code:{code}`, value = JSON `{"user_id": user_id, "access_token": access_token, "refresh_token": refresh_token}`, TTL = `settings.WXCODE_CODE_TTL` (30 seconds).
   - Return the code string.

   **exchange_wxcode_code(redis, code: str) -> dict | None:**
   - Use Redis GETDEL for atomic consumption: `raw = await redis.getdel(f"auth:wxcode_code:{code}")`.
   - If raw is None, return None (code expired, already used, or never existed).
   - Parse JSON, return dict with user_id, access_token, refresh_token.

   **get_redirect_url(db, user) -> str | None:**
   - Resolve the wxcode application URL for the user:
     1. If `user.last_used_tenant_id` is set, query `Tenant` by that ID and return `tenant.wxcode_url` if not None.
     2. Otherwise, query the user's most recent TenantMembership (ORDER BY created_at DESC LIMIT 1), load the Tenant, return `tenant.wxcode_url`.
     3. If no memberships or no wxcode_url configured, return None.
   - Import Tenant and TenantMembership lazily to avoid circular imports.

2. **CRITICAL: Wire session metadata from HTTP request into _issue_tokens for ALL auth flows.**

   This fulfills the locked decision for rich session metadata (IP address, device type, browser name/version, approximate city). Plan 01 added `user_agent` and `ip_address` keyword args to `_issue_tokens` — this step wires the actual values from the HTTP request.

   **In the login endpoint (auth/router.py):**
   - Extract metadata from the FastAPI `Request` object:
     ```python
     user_agent_str = request.headers.get("User-Agent")
     client_ip = request.client.host if request.client else None
     ```
   - Pass to `_issue_tokens` call:
     ```python
     result = await _issue_tokens(db, redis, user, user_agent=user_agent_str, ip_address=client_ip)
     ```
   - The login endpoint already receives `request: Request` as its first parameter (slowapi requirement from Phase 5).

   **In the MFA verify endpoint (auth/router.py):**
   - Same extraction pattern — extract `user_agent` and `ip_address` from `request`:
     ```python
     user_agent_str = request.headers.get("User-Agent")
     client_ip = request.client.host if request.client else None
     ```
   - Pass to `_issue_tokens` in the MFA verification flow:
     ```python
     result = await _issue_tokens(db, redis, user, user_agent=user_agent_str, ip_address=client_ip)
     ```

   **In the OAuth callback endpoint (auth/router.py):**
   - Same extraction and pass-through:
     ```python
     user_agent_str = request.headers.get("User-Agent")
     client_ip = request.client.host if request.client else None
     result = await _issue_tokens(db, redis, user, user_agent=user_agent_str, ip_address=client_ip)
     ```

   **In the token refresh endpoint (auth/router.py), if it calls _issue_tokens:**
   - Same pattern — extract and pass user_agent and ip_address so refreshed sessions also carry metadata.

   This ensures ALL UserSession rows have populated user_agent, device_type, browser_name, browser_version, ip_address, and city fields (city requires GEOLITE2_DB_PATH to be configured; gracefully null otherwise).
