  <action>
Add 4 service functions to admin/service.py under a new section "# Phase 22: Claude Provisioning":

Import `encrypt_value` from `wxcode_adm.common.crypto` at the top of the file.

1. `async def set_claude_token(db: AsyncSession, tenant_id: uuid.UUID, token: str, reason: str, actor_id: uuid.UUID) -> Tenant`:
   - Load tenant by ID — raise NotFoundError if not found
   - Encrypt the plaintext token: `encrypted = encrypt_value(token)`
   - Set `tenant.claude_oauth_token = encrypted`
   - Write audit: action="set_claude_token", resource_type="tenant", resource_id=str(tenant_id), details={"reason": reason} (do NOT log the token value)
   - Log: `logger.info("Admin set_claude_token: tenant_id=%s actor_id=%s", tenant_id, actor_id)`
   - Return tenant

2. `async def revoke_claude_token(db: AsyncSession, tenant_id: uuid.UUID, reason: str, actor_id: uuid.UUID) -> Tenant`:
   - Load tenant by ID — raise NotFoundError if not found
   - If tenant.claude_oauth_token is already None, raise ConflictError(error_code="NO_TOKEN", message="Tenant has no Claude token to revoke")
   - Set `tenant.claude_oauth_token = None`
   - Write audit: action="revoke_claude_token", resource_type="tenant", resource_id=str(tenant_id), details={"reason": reason}
   - Log and return tenant

3. `async def update_claude_config(db: AsyncSession, tenant_id: uuid.UUID, claude_default_model: str | None, claude_max_concurrent_sessions: int | None, claude_monthly_token_budget: int | None, actor_id: uuid.UUID) -> Tenant`:
   - Load tenant by ID — raise NotFoundError if not found
   - For each non-None param, update the corresponding tenant field. Special case: if claude_monthly_token_budget is 0, set to None (meaning unlimited)
   - Build a `changes` dict of what was actually updated (key: field name, value: new value) for the audit details
   - Write audit: action="update_claude_config", resource_type="tenant", resource_id=str(tenant_id), details=changes
   - Log and return tenant

4. `async def activate_tenant(db: AsyncSession, tenant_id: uuid.UUID, reason: str, actor_id: uuid.UUID) -> Tenant`:
   - Load tenant by ID — raise NotFoundError if not found
   - Validate: tenant.status must be "pending_setup" — raise ConflictError(error_code="INVALID_STATUS", message=f"Cannot activate tenant with status '{tenant.status}'. Only 'pending_setup' tenants can be activated.") if not
   - Validate: tenant.database_name must not be None — raise ConflictError(error_code="MISSING_DATABASE_NAME", message="Tenant must have a database_name configured before activation") if None
   - Set `tenant.status = "active"`
   - Write audit: action="activate_tenant", resource_type="tenant", resource_id=str(tenant_id), details={"reason": reason, "previous_status": "pending_setup"}
   - Log and return tenant

5. Update `get_tenant_detail()` to include Phase 20 fields in its return dict. Add these keys after the existing fields:
   - `"status": tenant.status`
   - `"database_name": tenant.database_name`
   - `"default_target_stack": tenant.default_target_stack`
   - `"neo4j_enabled": tenant.neo4j_enabled`
   - `"claude_default_model": tenant.claude_default_model`
   - `"claude_max_concurrent_sessions": tenant.claude_max_concurrent_sessions`
   - `"claude_monthly_token_budget": tenant.claude_monthly_token_budget`
   - `"has_claude_token": tenant.claude_oauth_token is not None`

All 4 new functions follow the same pattern as suspend_tenant: load tenant, validate state, mutate, audit, log, return.
