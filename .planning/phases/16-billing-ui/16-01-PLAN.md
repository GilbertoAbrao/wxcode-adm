TypeScript interfaces (matching backend schemas.py):

```typescript
interface BillingPlan {
  id: string;
  name: string;
  slug: string;
  monthly_fee_cents: number;
  token_quota: number;
  overage_rate_cents_per_token: number;
  member_cap: number;
  is_active: boolean;
  created_at: string;
  updated_at: string;
}

interface Subscription {
  id: string;
  tenant_id: string;
  plan: BillingPlan;
  status: string; // "free", "active", "past_due", "canceled", "trialing"
  current_period_start: string | null;
  current_period_end: string | null;
  tokens_used_this_period: number;
  created_at: string;
  updated_at: string;
}

interface CheckoutResponse {
  checkout_url: string;
  session_id: string;
}

interface PortalResponse {
  portal_url: string;
}
```

Export all interfaces.

Hooks to create:

1. `usePlans()` — `useQuery` GET `/billing/plans` (no X-Tenant-ID needed, any authenticated user). queryKey: `["billing", "plans"]`, staleTime: 5 minutes. Returns `BillingPlan[]`.

2. `useSubscription(tenantId: string | undefined)` — `useQuery` GET `/billing/subscription` with tenantHeaders(tenantId!). queryKey: `["billing", "subscription", tenantId]`, enabled: !!tenantId, staleTime: 30 seconds. Returns `Subscription`.

3. `useCreateCheckout(tenantId: string | undefined)` — `useMutation` POST `/billing/checkout` with body `{ plan_id }` and tenantHeaders(tenantId!). On success: does NOT invalidate queries (the redirect will leave the page). Returns `CheckoutResponse`.

4. `useCreatePortal(tenantId: string | undefined)` — `useMutation` POST `/billing/portal` with tenantHeaders(tenantId!). No body needed. Returns `PortalResponse`.

Copy the `tenantHeaders` helper from useTenant.ts (same pattern — private function within the file, not shared). Use `"use client"` directive at top.

Import apiClient from `@/lib/api-client` and `{ useMutation, useQuery, useQueryClient }` from `@tanstack/react-query`. Note: useQueryClient is imported but only needed if we add invalidation (Plan 16-02 will use it for polling); include the import anyway for Plan 16-02 readiness.
