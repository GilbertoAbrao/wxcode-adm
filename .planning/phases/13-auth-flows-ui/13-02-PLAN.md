1. Create `frontend/src/lib/validations.ts`:
   - Import { z } from "zod"
   - Export `emailSchema = z.string().email("Please enter a valid email address")`
   - Export `passwordSchema = z.string().min(8, "Password must be at least 8 characters").max(128, "Password must be at most 128 characters")`
   - Export `signupSchema = z.object({ email: emailSchema, password: passwordSchema })`
   - Export `loginSchema = z.object({ email: emailSchema, password: z.string().min(1, "Password is required") })`
   - Export `forgotPasswordSchema = z.object({ email: emailSchema })`
   - Export `resetPasswordSchema = z.object({ new_password: passwordSchema, confirm_password: z.string() }).refine(data => data.new_password === data.confirm_password, { message: "Passwords do not match", path: ["confirm_password"] })`
   - Export `verifyEmailSchema = z.object({ code: z.string().length(6, "Code must be 6 digits").regex(/^\d{6}$/, "Code must be 6 digits") })`
   - Export `workspaceSchema = z.object({ name: z.string().min(2, "Workspace name must be at least 2 characters").max(255, "Workspace name must be at most 255 characters") })`
   - Export `mfaCodeSchema = z.object({ code: z.string().min(6, "Code must be at least 6 characters").max(11, "Code must be at most 11 characters") })`
   - Export type aliases: `type SignupFormData = z.infer<typeof signupSchema>` etc.

     - Email: GlowInput leftIcon={Mail}
     - Password: GlowInput leftIcon={Lock}, rightIcon for show/hide

   - onSubmit handler — call loginMutation.mutate(data), onSuccess callback handles 3 branches:
     a. `response.mfa_required === true`:
        - If `response.mfa_setup_required`: redirect to `/mfa-setup?token=${response.mfa_token}` (MFA enrollment required by tenant but not yet set up — this is out of Phase 13 scope but route exists)
        - Else: redirect to `/mfa-verify?token=${encodeURIComponent(response.mfa_token!)}`
     b. `response.wxcode_redirect_url` is set:
        - Call `authContext.login({ access_token, refresh_token })` to store tokens
        - Then `window.location.href = \`${response.wxcode_redirect_url}?code=${response.wxcode_code}\``
     c. Normal login (no MFA, no wxcode redirect):
        - Call `authContext.login({ access_token, refresh_token })`
        - `router.push("/")`

   - Show API error messages below form:
     - 401: "Invalid email or password"
     - 403: "Please verify your email first" with a link to `/verify-email?email=${watchedEmail}`
     - Default: error.message

   - Footer links:
     - "Forgot password?" link to /forgot-password
     - "Don't have an account?" link to /signup

   - Styling: same card pattern as signup — max-w-sm, border-zinc-800, bg-obsidian-900/50, heading "Sign in to wxCode"
