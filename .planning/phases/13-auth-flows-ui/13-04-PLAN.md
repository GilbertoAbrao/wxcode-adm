   - Import useMfaVerify from @/hooks/useAuth
   - Import useAuthContext from @/providers/auth-provider
   - Import react-hook-form with zodResolver(mfaCodeSchema) from validations.ts
   - Import GlowButton, GlowInput from @/components/ui
   - Import { ShieldCheck, KeyRound } from lucide-react

   - Read mfa_token from searchParams.get("token") — if missing, redirect to /login

   - State:
     - `useBackupCode: boolean` (toggle between TOTP and backup code input)
     - `trustDevice: boolean` (checkbox state)

   - Heading: "Two-factor authentication"
   - Subtitle: changes based on mode:
     - TOTP mode: "Enter the 6-digit code from your authenticator app"
     - Backup mode: "Enter one of your backup codes"

   - Form:
     - Code input: GlowInput
       - TOTP mode: type="text", inputMode="numeric", maxLength={6}, placeholder="000000", leftIcon={ShieldCheck}, centered text (text-center tracking-widest text-lg font-mono)
       - Backup mode: type="text", maxLength={11}, placeholder="XXXXX-XXXXX", leftIcon={KeyRound}, centered text (text-center tracking-widest text-lg font-mono)
     - Trust device checkbox: native HTML checkbox + label "Trust this device for 30 days" — styled with accent-blue-500
     - Submit GlowButton "Verify"

   - Toggle link below form: "Use backup code instead" / "Use authenticator app instead" — toggles useBackupCode state and resets form

   - onSubmit:
     - Call mfaVerifyMutation.mutate({ mfa_token, code: data.code, trust_device: trustDevice })
     - onSuccess callback receives the response (from POST /auth/mfa/verify which returns { access_token, refresh_token, wxcode_redirect_url?, wxcode_code? }):
       a. If wxcode_redirect_url is present:
          - Call authContext.login({ access_token, refresh_token })
          - window.location.href = `${wxcode_redirect_url}?code=${wxcode_code}`
       b. Else:
          - Call authContext.login({ access_token, refresh_token })
          - router.push("/")
     - onError: show API error inline ("Invalid code", "Code expired", etc.)

   - NOTE: The useMfaVerify hook's mutation function needs to return the full response body (not just message), since we need access_token, refresh_token, and wxcode fields. Ensure the hook in useAuth.ts returns the full response. If the hook from 13-01 only returns a generic MessageResponse, update the response type to match the actual backend response (which returns access_token, refresh_token, token_type, wxcode_redirect_url?, wxcode_code?). The hook should use type `MfaVerifyResponse { access_token: string; refresh_token: string; token_type: string; wxcode_redirect_url?: string; wxcode_code?: string }`.

   - Styling: same card pattern (max-w-sm, border-zinc-800, bg-obsidian-900/50)
  </action>
