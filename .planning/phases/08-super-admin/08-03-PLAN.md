    2. Update `backend/src/wxcode_adm/admin/service.py` — add user management functions:

       a. `search_users(db, limit=20, offset=0, q=None, tenant_id=None) -> tuple[list[dict], int]`:
          - Base query: `select(User)`.
          - If `q` (search string): filter with `or_(User.email.ilike(f"%{q}%"), User.display_name.ilike(f"%{q}%"))`. Import `or_` from sqlalchemy.
          - If `tenant_id`: join TenantMembership, filter `TenantMembership.tenant_id == tenant_id`.
          - Count query with same filters.
          - Order by `User.created_at.desc()`, apply limit/offset.
          - Return `(items_as_dicts, total)`.

       b. `get_user_detail(db, user_id) -> dict`:
          - Load User by ID. Raise NotFoundError if not found.
          - Load memberships: `select(TenantMembership).where(TenantMembership.user_id == user_id)` with `selectinload` or join on Tenant for tenant name/slug. For each membership, include `is_blocked` (use `getattr(m, 'is_blocked', False)` for safety before migration).
          - Load sessions: `select(UserSession).where(UserSession.user_id == user_id)`.
          - Return dict matching UserDetailResponse shape.

       c. `block_user(db, redis, user_id, tenant_id, reason, actor_id) -> None`:
          - Load TenantMembership by user_id + tenant_id. Raise NotFoundError(error_code="MEMBERSHIP_NOT_FOUND") if not found.
          - Set `membership.is_blocked = True`.
          - Per research recommendation: block at get_tenant_context level (already done in Plan 01) — no JTI blacklisting needed for per-tenant block. The membership check happens every request so enforcement is immediate.
          - write_audit: action="block_user", resource_type="user", resource_id=str(user_id), tenant_id=tenant_id, details={"reason": reason, "tenant_id": str(tenant_id)}.

       d. `unblock_user(db, user_id, tenant_id, reason, actor_id) -> None`:
          - Load TenantMembership. Raise NotFoundError if not found.
          - Set `membership.is_blocked = False`.
          - write_audit: action="unblock_user", resource_type="user", resource_id=str(user_id), tenant_id=tenant_id, details={"reason": reason, "tenant_id": str(tenant_id)}.

       e. `force_password_reset(db, redis, user_id, reason, actor_id) -> None`:
          - Load User by ID. Raise NotFoundError if not found.
          - Set `user.password_reset_required = True` (flag enforced by get_current_user from Plan 01).
          - Invalidate all sessions: query UserSession.access_token_jti for this user, call `blacklist_jti(redis, jti)` for each. Delete all RefreshToken rows for this user.
          - Send reset email: enqueue arq job. Use lazy import: `from wxcode_adm.tasks.worker import get_arq_pool`. Create arq pool and enqueue `send_reset_email` job with user's email. Use the existing `generate_reset_token` from `wxcode_adm.auth.service` to create a reset token, then enqueue `send_reset_email` with it.
          - write_audit: action="force_password_reset", resource_type="user", resource_id=str(user_id), details={"reason": reason}.
          - Note: When user completes password reset (existing POST /auth/reset-password), they must also clear `password_reset_required = False`. This needs a one-line addition to `auth/service.py:reset_password()` — add `user.password_reset_required = False` after setting the new password hash (use `setattr` with hasattr guard for safety before migration).

    3. **Critical:** Update `backend/src/wxcode_adm/auth/service.py` — in the `reset_password` function, after setting `user.password_hash`, add: `if hasattr(user, 'password_reset_required'): user.password_reset_required = False`. This clears the forced-reset flag when the user completes their password reset.
