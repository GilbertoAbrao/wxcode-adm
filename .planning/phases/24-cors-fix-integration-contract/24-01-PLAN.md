In `create_app()` in main.py, replace the current CORS middleware block:

```python
app.add_middleware(
    CORSMiddleware,
    allow_origin_regex=r".*",
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

With a production-ready implementation:

1. Create a helper function `_build_cors_origins()` inside main.py (or as a local in create_app) that:
   - Starts with `settings.ALLOWED_ORIGINS` (the static list from config, e.g. `["http://localhost:3060"]`)
   - Adds `settings.FRONTEND_URL` if not already in the list (safety net)
   - Returns the combined list

2. For dynamic tenant wxcode_url origins, implement a custom CORS middleware approach. Since tenant wxcode_urls can change at runtime and FastAPI's CORSMiddleware only accepts a static list at init time, use a **startup event to preload tenant origins + `allow_origin_regex` as fallback for the static list**. Specifically:

   **Approach: Use `is_origin_allowed` callback via subclassing.** Subclass `CORSMiddleware` to override origin checking:

   ```python
   class DynamicCORSMiddleware(CORSMiddleware):
       """CORS middleware that checks static origins + tenant wxcode_urls from DB cache."""

       def __init__(self, app, tenant_origins_loader, **kwargs):
           super().__init__(app, **kwargs)
           self._tenant_origins_loader = tenant_origins_loader

       def is_allowed_origin(self, origin: str) -> bool:
           # Check static origins first (parent class behavior)
           if super().is_allowed_origin(origin):
               return True
           # Check dynamic tenant origins
           return origin in self._tenant_origins_loader()
   ```

   Create a module-level `_tenant_origin_cache: set[str]` that is populated during lifespan startup and can be refreshed. In lifespan startup (after DB is verified), query all non-null `Tenant.wxcode_url` values and populate the cache:

   ```python
   async with async_session_maker() as session:
       result = await session.execute(
           select(Tenant.wxcode_url).where(Tenant.wxcode_url.isnot(None))
       )
       _tenant_origin_cache.update(row[0] for row in result.all())
   ```

   Pass `lambda: _tenant_origin_cache` as the `tenant_origins_loader` to DynamicCORSMiddleware.

3. Replace the `app.add_middleware(CORSMiddleware, ...)` call with:

   ```python
   app.add_middleware(
       DynamicCORSMiddleware,
       tenant_origins_loader=lambda: _tenant_origin_cache,
       allow_origins=_build_cors_origins(),
       allow_credentials=True,
       allow_methods=["*"],
       allow_headers=["*"],
   )
   ```

4. Remove `allow_origin_regex=r".*"` entirely — it is the security hole being fixed.

**Important:** The `DynamicCORSMiddleware` class and `_tenant_origin_cache` should be defined at module level in main.py (above create_app). Keep it simple — no separate file needed for ~20 lines of code.

**Important:** The Starlette CORSMiddleware `is_allowed_origin` method signature must be matched exactly. Check the Starlette source — in recent versions the method is `is_allowed_origin(self, origin: str) -> bool`. If the method name differs, adjust accordingly.
