1. Create `backend/src/wxcode_adm/tenants/dependencies.py` with three dependencies:

**get_tenant_context(x_tenant_id, user, db) -> tuple[Tenant, TenantMembership]:**
- `x_tenant_id: Annotated[str | None, Header()] = None` — FastAPI auto-converts underscores to hyphens, so this maps to the `X-Tenant-Id` / `X-Tenant-ID` header (case-insensitive in HTTP)
- `user: User = Depends(require_verified)` — chain from auth dependencies
- `db: AsyncSession = Depends(get_session)`
- If x_tenant_id is None: raise NoTenantContextError()
- Try parsing x_tenant_id as UUID first; if ValueError, treat as slug
- Query Tenant by id or slug
- If tenant not found: raise TenantNotFoundError()
- Query TenantMembership where tenant_id == tenant.id AND user_id == user.id
- If membership not found: raise TenantNotFoundError() (same error -- don't leak tenant existence)
- Return (tenant, membership) tuple
- IMPORTANT: Tenant and TenantMembership are NOT TenantModel subclasses, so no execution_options needed

**require_role(minimum_role: MemberRole) -> dependency function:**
- This is a dependency FACTORY (returns a closure) -- standard FastAPI pattern
- The inner function takes `ctx: tuple[Tenant, TenantMembership] = Depends(get_tenant_context)`
- Checks `membership.role.level >= minimum_role.level` (integer comparison)
- If insufficient: raise InsufficientRoleError(message=f"Requires {minimum_role.value} role or above")
- Returns (tenant, membership) tuple
- Usage: `Depends(require_role(MemberRole.ADMIN))`

**require_tenant_member(ctx) -> tuple[Tenant, TenantMembership]:**
- Simple passthrough of get_tenant_context -- any authenticated member passes
- Exists for semantic clarity: some endpoints need "any member" not a specific role
- `ctx: tuple[Tenant, TenantMembership] = Depends(get_tenant_context)` -> return ctx

2. Create `backend/src/wxcode_adm/tenants/service.py` with:

**generate_unique_slug(db: AsyncSession, name: str) -> str:**
- `from slugify import slugify`
- `base_slug = slugify(name, max_length=80)` — 80 char base to leave room for counter suffix
- If base_slug is empty after slugify (e.g., all special chars): base_slug = "workspace"
- Loop: check if slug is taken via `select(Tenant).where(Tenant.slug == slug)`
- If not taken, return slug
- If taken, try `{base_slug}-{counter}` with counter starting at 2
- Cap at 10 iterations; if all taken, raise ConflictError("SLUG_UNAVAILABLE", "Workspace name is too common, try a different one")
- The UNIQUE constraint on tenants.slug is the authoritative guard; this pre-check loop handles the common case efficiently

**create_workspace(db: AsyncSession, user: User, name: str) -> tuple[Tenant, TenantMembership]:**
- Generate unique slug from name
- Create Tenant(name=name, slug=slug)
- db.add(tenant), db.flush() to get tenant.id
- Create TenantMembership(user_id=user.id, tenant_id=tenant.id, role=MemberRole.OWNER, billing_access=True)
  Per user decision: creator gets Owner role with billing_access=True by default
- db.add(membership), db.flush()
- Return (tenant, membership)

**get_user_tenants(db: AsyncSession, user_id: uuid.UUID) -> list[dict]:**
- Select TenantMembership joined with Tenant where user_id == user_id
- Return list of dicts: {id, name, slug, role, billing_access} for each membership
- Use selectinload or a join query for efficiency
