**upgrade() function creates 4 tables in dependency order:**

1. **tenants** table (no FKs, create first):
   - id: UUID, server_default=gen_random_uuid(), primary key
   - created_at: DateTime(timezone=True), server_default=now()
   - updated_at: DateTime(timezone=True), server_default=now()
   - name: String(255), nullable=False
   - slug: String(100), nullable=False, unique
   - PrimaryKeyConstraint with name following naming convention: "pk_tenants"
   - UniqueConstraint on slug: "uq_tenants_slug"
   - Index on slug: "ix_tenants_slug" (unique=True)

2. **tenant_memberships** table (FKs to users + tenants):
   - id: UUID, server_default=gen_random_uuid(), primary key
   - created_at, updated_at (same pattern)
   - user_id: UUID, ForeignKey("users.id", ondelete="CASCADE"), nullable=False
   - tenant_id: UUID, ForeignKey("tenants.id", ondelete="CASCADE"), nullable=False
   - role: String(20), nullable=False (NOT native enum -- per research decision)
   - billing_access: Boolean, default=False, nullable=False
   - invited_by_id: UUID, ForeignKey("users.id", ondelete="SET NULL"), nullable=True
   - PrimaryKeyConstraint: "pk_tenant_memberships"
   - UniqueConstraint on (user_id, tenant_id): "uq_tenant_memberships_user_tenant"
   - Indexes on user_id and tenant_id: "ix_tenant_memberships_user_id", "ix_tenant_memberships_tenant_id"
   - ForeignKeyConstraints with naming convention names

3. **invitations** table (FKs to tenants + users):
   - id: UUID, server_default=gen_random_uuid(), primary key
   - created_at, updated_at (same pattern)
   - email: String(320), nullable=False
   - tenant_id: UUID, ForeignKey("tenants.id", ondelete="CASCADE"), nullable=False
   - role: String(20), nullable=False
   - billing_access: Boolean, default=False, nullable=False
     This column carries the billing_access intent from the invite API call. It is read by
     accept_invitation and auto_join_pending_invitations to set TenantMembership.billing_access
     on the resulting membership row.
   - token_hash: String(64), nullable=False
   - invited_by_id: UUID, ForeignKey("users.id", ondelete="CASCADE"), nullable=False
   - expires_at: DateTime(timezone=True), nullable=False
   - accepted_at: DateTime(timezone=True), nullable=True
   - PrimaryKeyConstraint: "pk_invitations"
   - UniqueConstraint on (email, tenant_id): "uq_invitations_email_tenant"
   - Index on email: "ix_invitations_email"
   - Index on tenant_id: "ix_invitations_tenant_id"

4. **ownership_transfers** table (FKs to tenants + users):
   - id: UUID, server_default=gen_random_uuid(), primary key
   - created_at, updated_at (same pattern)
   - tenant_id: UUID, ForeignKey("tenants.id", ondelete="CASCADE"), nullable=False, unique (one pending per tenant)
   - from_user_id: UUID, ForeignKey("users.id", ondelete="CASCADE"), nullable=False
   - to_user_id: UUID, ForeignKey("users.id", ondelete="CASCADE"), nullable=False
   - expires_at: DateTime(timezone=True), nullable=False
   - PrimaryKeyConstraint: "pk_ownership_transfers"
   - UniqueConstraint on tenant_id: "uq_ownership_transfers_tenant_id"
   - Index on tenant_id: "ix_ownership_transfers_tenant_id"

**downgrade() function drops tables in reverse order:**
- drop ownership_transfers, invitations, tenant_memberships, tenants
