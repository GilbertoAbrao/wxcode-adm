2. Create `backend/src/wxcode_adm/tenants/models.py` with the following models:

**MemberRole** (Python enum.Enum):
- OWNER = "owner", ADMIN = "admin", DEVELOPER = "developer", VIEWER = "viewer"
- Add a `level` property returning integer: owner=4, admin=3, developer=2, viewer=1
- This enables `membership.role.level >= required.level` comparisons in require_role

**Tenant** (TimestampMixin, Base):
- `__tablename__ = "tenants"`
- `name: Mapped[str]` — String(255), nullable=False
- `slug: Mapped[str]` — String(100), unique=True, index=True, nullable=False
- `memberships` relationship to TenantMembership (back_populates="tenant", cascade="all, delete-orphan")
- Tenant inherits Base+TimestampMixin, NOT TenantModel (it IS the tenant, not scoped to one)

**TenantMembership** (TimestampMixin, Base):
- `__tablename__ = "tenant_memberships"`
- `__table_args__` with UniqueConstraint("user_id", "tenant_id", name="uq_tenant_memberships_user_tenant")
- `user_id: Mapped[uuid.UUID]` — ForeignKey("users.id", ondelete="CASCADE"), nullable=False, index=True
- `tenant_id: Mapped[uuid.UUID]` — ForeignKey("tenants.id", ondelete="CASCADE"), nullable=False, index=True
- `role: Mapped[MemberRole]` — Enum(MemberRole, native_enum=False, length=20), nullable=False
  CRITICAL: use `native_enum=False` to avoid PostgreSQL CREATE TYPE and Alembic migration issues (see research pitfall #1)
- `billing_access: Mapped[bool]` — Boolean, default=False, nullable=False
  Per user decision: billing is a permission toggle, NOT a role
- `invited_by_id: Mapped[uuid.UUID | None]` — ForeignKey("users.id", ondelete="SET NULL"), nullable=True
- `user` relationship (foreign_keys=[user_id], back_populates="memberships")
- `tenant` relationship (back_populates="memberships")

**Invitation** (TimestampMixin, Base):
- `__tablename__ = "invitations"`
- `__table_args__` with UniqueConstraint("email", "tenant_id", name="uq_invitations_email_tenant")
- `email: Mapped[str]` — String(320), nullable=False, index=True
- `tenant_id: Mapped[uuid.UUID]` — ForeignKey("tenants.id", ondelete="CASCADE"), nullable=False, index=True
- `role: Mapped[MemberRole]` — Enum(MemberRole, native_enum=False, length=20), nullable=False
- `token_hash: Mapped[str]` — String(64), nullable=False (SHA-256 hex digest of the itsdangerous token)
- `invited_by_id: Mapped[uuid.UUID]` — ForeignKey("users.id", ondelete="CASCADE"), nullable=False
- `expires_at: Mapped[datetime]` — DateTime(timezone=True), nullable=False
- `accepted_at: Mapped[datetime | None]` — DateTime(timezone=True), nullable=True, default=None
- `tenant` relationship (lazy="joined" for efficient access in accept flow)
- Invitations are stored in DB per research recommendation — the itsdangerous token is the delivery mechanism, the DB record is the authoritative state

**OwnershipTransfer** (TimestampMixin, Base):
- `__tablename__ = "ownership_transfers"`
- `tenant_id: Mapped[uuid.UUID]` — ForeignKey("tenants.id", ondelete="CASCADE"), nullable=False, unique=True, index=True (only one pending transfer per tenant)
- `from_user_id: Mapped[uuid.UUID]` — ForeignKey("users.id", ondelete="CASCADE"), nullable=False
- `to_user_id: Mapped[uuid.UUID]` — ForeignKey("users.id", ondelete="CASCADE"), nullable=False
- `expires_at: Mapped[datetime]` — DateTime(timezone=True), nullable=False
- No separate accept token needed — the to_user authenticates via JWT and the endpoint verifies identity + membership

**NoTenantContextError** (ForbiddenError):
- error_code="TENANT_CONTEXT_REQUIRED"
- message="This endpoint requires a tenant context. Include X-Tenant-ID header or complete workspace setup at POST /api/v1/onboarding/workspace."
- status_code=403
- No-arg constructor (consistent with auth exceptions pattern)

**TenantNotFoundError** (NotFoundError):
- error_code="TENANT_NOT_FOUND"
- message="Tenant not found or you are not a member"
- status_code=404
- Per research: same message for "doesn't exist" and "not a member" to prevent leaking info
- No-arg constructor

**InsufficientRoleError** (ForbiddenError):
- error_code="INSUFFICIENT_ROLE"
- message parameter accepted (default: "You do not have the required role for this action")
- status_code=403
- Constructor takes optional message kwarg

**NotMemberError** (ForbiddenError):
- error_code="NOT_MEMBER"
- message="You are not a member of this tenant"
- status_code=403
- No-arg constructor

**OwnerCannotSelfDemoteError** (ForbiddenError):
- error_code="OWNER_CANNOT_SELF_DEMOTE"
- message="Transfer ownership first before changing your own role"
- status_code=403
- No-arg constructor

**OwnerCannotLeaveError** (ForbiddenError):
- error_code="OWNER_CANNOT_LEAVE"
- message="Transfer ownership before leaving the tenant"
- status_code=403
- No-arg constructor

**InvitationAlreadyExistsError** (ConflictError):
- error_code="INVITATION_ALREADY_EXISTS"
- message="An active invitation already exists for this email in this tenant"
- status_code=409
- No-arg constructor

**AlreadyMemberError** (ConflictError):
- error_code="ALREADY_MEMBER"
- message="User is already a member of this tenant"
- status_code=409
- No-arg constructor

**TransferAlreadyPendingError** (ConflictError):
- error_code="TRANSFER_ALREADY_PENDING"

- status_code=409
- No-arg constructor
