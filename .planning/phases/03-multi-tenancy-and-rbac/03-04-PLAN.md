Add the following service functions to `backend/src/wxcode_adm/tenants/service.py`:

**change_role(db: AsyncSession, tenant: Tenant, actor: TenantMembership, target_user_id: uuid.UUID, new_role: MemberRole, billing_access: bool | None) -> TenantMembership:**
- Guard ORDER matters -- check owner self-demotion FIRST (per research pitfall #5):
  1. Find target membership: query TenantMembership where user_id == target_user_id AND tenant_id == tenant.id. If not found, raise NotFoundError("MEMBER_NOT_FOUND", "Member not found in this tenant")
  2. If target_membership.role == MemberRole.OWNER AND target_membership.user_id == actor.user_id: raise OwnerCannotSelfDemoteError() — Owner cannot demote themselves
  3. If target_membership.role == MemberRole.OWNER AND actor.role != MemberRole.OWNER: raise InsufficientRoleError("Only the Owner can change the Owner's role") — only Owner can touch another Owner (edge case: there's only ever one Owner, so this guards against Admin trying to change Owner)
  4. Actor cannot assign OWNER role via this endpoint — ownership is transferred separately. If new_role == MemberRole.OWNER: raise ForbiddenError("USE_TRANSFER", "Use ownership transfer to assign Owner role")
  5. Update target_membership.role = new_role
  6. If billing_access is not None: target_membership.billing_access = billing_access
  7. Return target_membership

**remove_member(db: AsyncSession, tenant: Tenant, actor: TenantMembership, target_user_id: uuid.UUID) -> None:**
- Cannot remove yourself via this endpoint — use leave_tenant for that
  If target_user_id == actor.user_id: raise ForbiddenError("USE_LEAVE", "Use the leave endpoint to remove yourself")
- Find target membership: query TenantMembership where user_id == target_user_id AND tenant_id == tenant.id. If not found, raise NotFoundError("MEMBER_NOT_FOUND", "Member not found")
- Cannot remove the Owner: if target_membership.role == MemberRole.OWNER: raise ForbiddenError("CANNOT_REMOVE_OWNER", "Cannot remove the tenant Owner")
- Delete the membership: await db.delete(target_membership)
- Per user decision: account persists, member just loses this tenant membership

**leave_tenant(db: AsyncSession, tenant_id: uuid.UUID, user_id: uuid.UUID) -> None:**
- Find membership: query TenantMembership where user_id == user_id AND tenant_id == tenant_id. If not found, raise NotMemberError()
- If membership.role == MemberRole.OWNER: raise OwnerCannotLeaveError() — Owner must transfer first
- Delete the membership: await db.delete(membership)

**initiate_transfer(db: AsyncSession, tenant: Tenant, owner_membership: TenantMembership, to_user_id: uuid.UUID) -> OwnershipTransfer:**
- Verify actor IS the Owner: if owner_membership.role != MemberRole.OWNER: raise InsufficientRoleError("Only the Owner can initiate ownership transfer")
- Cannot transfer to yourself: if to_user_id == owner_membership.user_id: raise ForbiddenError("CANNOT_TRANSFER_TO_SELF", "Cannot transfer ownership to yourself")
- Verify target is a member: query TenantMembership where user_id == to_user_id AND tenant_id == tenant.id. If not found, raise NotFoundError("MEMBER_NOT_FOUND", "Target user is not a member of this tenant")
- Check for existing pending transfer: query OwnershipTransfer where tenant_id == tenant.id. If found:
  - If not expired (expires_at > now()): raise TransferAlreadyPendingError()
  - If expired: delete the stale record and proceed
- Create OwnershipTransfer: tenant_id=tenant.id, from_user_id=owner_membership.user_id, to_user_id=to_user_id, expires_at=datetime.now(tz=timezone.utc) + timedelta(days=7)
- db.add(transfer), db.flush()
- Return transfer

**accept_transfer(db: AsyncSession, tenant_id: uuid.UUID, user_id: uuid.UUID) -> None:**
- Find the pending transfer: query OwnershipTransfer where tenant_id == tenant_id AND to_user_id == user_id. If not found, raise NotFoundError("TRANSFER_NOT_FOUND", "No pending ownership transfer found for you")
- Check expiry: if transfer.expires_at < now(), delete it and raise TokenExpiredError()
- Find from_user membership (current Owner): query TenantMembership where user_id == transfer.from_user_id AND tenant_id == tenant_id
- Find to_user membership (new Owner): query TenantMembership where user_id == user_id AND tenant_id == tenant_id
- Swap: from_user_membership.role = MemberRole.ADMIN (downgrade), to_user_membership.role = MemberRole.OWNER (upgrade)
- Delete the transfer record: await db.delete(transfer)

**get_pending_transfer(db: AsyncSession, tenant_id: uuid.UUID) -> OwnershipTransfer | None:**
- Query OwnershipTransfer where tenant_id == tenant_id
- If found and not expired, return it. If expired, delete and return None. If not found, return None.

All datetime comparisons must handle timezone-naive (SQLite in tests) vs timezone-aware (PostgreSQL in production):
```python
expires_at = transfer.expires_at
if expires_at.tzinfo is None:
    expires_at = expires_at.replace(tzinfo=timezone.utc)
```
This follows the same pattern used in auth/service.py for refresh token expiry.

Add these endpoints to `backend/src/wxcode_adm/tenants/router.py`:

**PATCH /api/v1/tenants/current/members/{user_id}/role** (change member's role):
- Path param: user_id: UUID
- Body: ChangeRoleRequest (role: str, billing_access: bool | None = None)
- Depends: ctx = Depends(require_role(MemberRole.ADMIN))
- Calls service.change_role(db, tenant, membership, user_id, MemberRole(body.role), body.billing_access)
- Returns MembershipResponse (with updated role info)

**DELETE /api/v1/tenants/current/members/{user_id}** (remove member):
- Path param: user_id: UUID
- Depends: ctx = Depends(require_role(MemberRole.ADMIN))
- Calls service.remove_member(db, tenant, membership, user_id)
- Returns MessageResponse("Member removed from tenant")

**POST /api/v1/tenants/current/leave** (leave tenant voluntarily):
- No body needed
- Depends: ctx = Depends(require_tenant_member) — any member can leave
- Calls service.leave_tenant(db, tenant.id, membership.user_id)
- Returns MessageResponse("You have left the tenant")

**POST /api/v1/tenants/current/transfer** (initiate ownership transfer):
- Body: InitiateTransferRequest (to_user_id: UUID)
- Depends: ctx = Depends(require_role(MemberRole.OWNER)) — ONLY Owner can initiate
- Calls service.initiate_transfer(db, tenant, membership, body.to_user_id)
- Returns TransferResponse with status 201

**POST /api/v1/tenants/current/transfer/accept** (accept ownership transfer):
- No body needed — the authenticated user's identity is the verification
- Depends: ctx = Depends(require_tenant_member) — any member who receives a transfer can accept
- Calls service.accept_transfer(db, tenant.id, membership.user_id)
- Returns MessageResponse("Ownership transferred successfully")

**GET /api/v1/tenants/current/transfer** (get pending transfer):
- Depends: ctx = Depends(require_role(MemberRole.OWNER)) — Owner can check transfer status
- Calls service.get_pending_transfer(db, tenant.id)
- Returns TransferResponse or 404 if no pending transfer

For the MembershipResponse returned by change_role, the response needs the user's email. Either:
- Join TenantMembership with User in the service function and return both
- Or in the router, look up the user email from db after the role change

Prefer the simpler approach: the service function returns the updated membership, and the router constructs MembershipResponse from it by additionally querying the user's email if needed.
