<task type="auto">
  <name>Task 1: Dockerfile, .dockerignore, and docker-compose.yml</name>
  <files>
    backend/Dockerfile
    backend/.dockerignore
    docker-compose.yml
  </files>
  <action>
    **backend/Dockerfile:**
    - Use `python:3.11-slim` as base (not alpine — asyncpg needs gcc for building)
    - Set WORKDIR /app
    - Copy pyproject.toml first (for layer caching)
    - RUN pip install --no-cache-dir -e "." (installs all dependencies)
    - Copy src/ and alembic/ directories
    - Copy alembic.ini
    - EXPOSE 8060
    - CMD: `uvicorn wxcode_adm.main:app --host 0.0.0.0 --port 8060`
    - NOTE: The docker-compose.yml overrides CMD for dev (adds --reload). The Dockerfile CMD is the production default.

    **backend/.dockerignore:**
    - __pycache__, *.pyc, .pytest_cache, .git, .env, *.egg-info, .venv, node_modules, .mypy_cache

    **docker-compose.yml (at repo root):**

    Service: **postgres**
    - image: postgres:17-alpine
    - environment: POSTGRES_USER=wxcode, POSTGRES_PASSWORD=wxcode_dev, POSTGRES_DB=wxcode_adm
    - ports: "5432:5432"
    - volumes: postgres_data:/var/lib/postgresql/data
    - healthcheck: pg_isready -U wxcode -d wxcode_adm (interval 5s, timeout 5s, retries 10, start_period 10s)

    Service: **redis**
    - image: redis:7-alpine
    - ports: "6379:6379"
    - healthcheck: redis-cli ping (interval 5s, timeout 5s, retries 10)

    Service: **api**
    - build: context: ./backend
    - command: `sh -c "alembic upgrade head && uvicorn wxcode_adm.main:app --host 0.0.0.0 --port 8060 --reload"`
    - ports: "8060:8060"
    - env_file: ./backend/.env
    - environment (override for Docker networking):
      DATABASE_URL: postgresql+asyncpg://wxcode:wxcode_dev@postgres:5432/wxcode_adm
      REDIS_URL: redis://redis:6379/0
    - depends_on: postgres (condition: service_healthy), redis (condition: service_healthy)
    - volumes: ./backend/src:/app/src (hot-reload source mount)

    Service: **worker**
    - build: context: ./backend
    - command: arq wxcode_adm.tasks.worker.WorkerSettings
    - env_file: ./backend/.env
    - environment (override for Docker networking):
      DATABASE_URL: postgresql+asyncpg://wxcode:wxcode_dev@postgres:5432/wxcode_adm
      REDIS_URL: redis://redis:6379/0
    - depends_on: postgres (condition: service_healthy), redis (condition: service_healthy)

    Volumes: postgres_data (named volume)

    NOTE: The api service overrides DATABASE_URL and REDIS_URL in environment (Docker hostnames postgres/redis instead of localhost). This takes precedence over the .env file values.
    NOTE: arq --watch flag from research is optional (it reloads worker on file changes). Include it only if arq supports it; if not, omit it.
  </action>
  <verify>
    Run `docker compose config` from repo root — must parse without errors.
    Run `docker compose build` — must build the backend image successfully.
    Verify Dockerfile builds: `docker build -t wxcode-adm-test ./backend` — must succeed.
  </verify>
  <done>Dockerfile builds a working Python 3.11 image with all dependencies. docker-compose.yml defines 4 services with proper healthchecks and dependency ordering. .dockerignore excludes build artifacts and secrets.</done>
