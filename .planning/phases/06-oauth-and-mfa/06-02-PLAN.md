<task type="auto">
  <name>Task 1: MFA enrollment service functions</name>
  <files>
    backend/src/wxcode_adm/auth/service.py
  </files>
  <action>
    Add MFA enrollment service functions to auth/service.py:

    1. **generate_backup_codes()** — Pure function (no DB):
       - Generate 10 backup codes using `secrets.token_urlsafe(8)[:10].upper()` for each
       - Format each code as "XXXXX-XXXXX" (5 chars, dash, 5 chars) for display
       - Hash each code (without the dash — strip dashes before hashing) using `hash_password` from auth/password.py (argon2id)
       - Return tuple: `(plaintext_formatted: list[str], hashed: list[str])`
       - Use constant `BACKUP_CODE_COUNT = 10`

    2. **generate_qr_code_base64(uri: str) -> str** — Pure function:
       - Import qrcode and io, base64
       - `img = qrcode.make(uri)`
       - Save to BytesIO as PNG
       - Return base64 encoded string

    3. **mfa_begin_enrollment(db: AsyncSession, user: User) -> dict** — Begin MFA enrollment:
       - If user.mfa_enabled is True, raise AppError("MFA_ALREADY_ENABLED", "MFA is already enabled", 400)
       - Generate a new TOTP secret: `secret = pyotp.random_base32()`
       - Store the secret on user.mfa_secret (temporary — not yet confirmed)
       - Generate provisioning URI: `pyotp.TOTP(secret).provisioning_uri(name=user.email, issuer_name="WXCODE")`
       - Generate QR code base64 from the URI
       - Return dict with keys: secret, qr_code, provisioning_uri
       - Do NOT set mfa_enabled=True yet — that happens on confirm

    4. **mfa_confirm_enrollment(db: AsyncSession, user: User, code: str) -> list[str]** — Confirm enrollment:
       - If user.mfa_secret is None, raise AppError("MFA_NOT_STARTED", "Begin enrollment first", 400)
       - If user.mfa_enabled is True, raise AppError("MFA_ALREADY_ENABLED", "MFA is already enabled", 400)
       - Verify the TOTP code: `pyotp.TOTP(user.mfa_secret).verify(code, valid_window=1)`
       - If invalid, raise MfaInvalidCodeError()
       - Set user.mfa_enabled = True
       - Generate backup codes: plaintext, hashed = generate_backup_codes()
       - Delete any existing MfaBackupCode rows for this user (cleanup from previous failed enrollments)
       - Create MfaBackupCode rows: one per hashed code, user_id=user.id, code_hash=hash, used_at=None
       - db.flush() to persist
       - Return plaintext backup codes (shown to user ONCE and never stored)

    5. **mfa_disable(db: AsyncSession, redis: Redis, user: User, code: str) -> None** — Disable MFA:
       - If not user.mfa_enabled, raise AppError("MFA_NOT_ENABLED", "MFA is not enabled", 400)
       - Try TOTP verification first: `pyotp.TOTP(user.mfa_secret).verify(code, valid_window=1)`
       - If TOTP fails, try backup code: strip dashes from `code`, query MfaBackupCode rows for user where used_at IS NULL, iterate and verify_password(code_stripped, row.code_hash). If match found, mark row.used_at = now.
       - If neither TOTP nor backup code matches, raise MfaInvalidCodeError()
       - Set user.mfa_enabled = False, user.mfa_secret = None
       - Delete all MfaBackupCode rows for this user
       - Per locked decision: "if tenant enforcement is on, they will be re-prompted to enroll on next login"

    Import pyotp at the top of the file. Import MfaBackupCode from auth/models.py. Import MfaInvalidCodeError from auth/exceptions.py. Import qrcode, io, base64 for QR generation.
  </action>

  <action>
    Add MFA enrollment routes to `auth_api_router` in auth/router.py:

    1. **POST /auth/mfa/enroll** — Begin MFA enrollment:
       - Requires authentication: `user: User = Depends(require_verified)`
       - Calls `service.mfa_begin_enrollment(db, user)`
       - Returns `MfaEnrollBeginResponse`
       - Audit: write_audit action="mfa_enroll_begin"
       - No rate limiting (authenticated endpoint)

    2. **POST /auth/mfa/confirm** — Confirm MFA enrollment:
       - Requires authentication: `user: User = Depends(require_verified)`
       - Accept `MfaEnrollConfirmRequest` body (code: str)
       - Calls `service.mfa_confirm_enrollment(db, user, body.code)`
       - Returns `MfaEnrollConfirmResponse(backup_codes=result)`
       - Audit: write_audit action="mfa_enroll_confirm"
       - IMPORTANT: backup_codes are shown to the user ONCE in this response and never stored in plaintext

    3. **DELETE /auth/mfa** — Disable MFA:
       - Requires authentication: `user: User = Depends(require_verified)`
       - Accept `MfaDisableRequest` body (code: str)
       - Calls `service.mfa_disable(db, redis, user, body.code)`
       - Returns `MessageResponse(message="MFA has been disabled")`
       - Audit: write_audit action="mfa_disable"

    4. **GET /auth/mfa/status** — Check MFA status:
       - Requires authentication: `user: User = Depends(require_verified)`
       - Returns `MfaStatusResponse(mfa_enabled=user.mfa_enabled)`
       - No audit (read-only)

    All routes use `request: Request` as first parameter (slowapi convention even when not rate-limited — consistent pattern). Import the new schemas and service functions.
