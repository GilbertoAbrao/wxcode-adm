  <action>
    Modify and add service functions in auth/service.py:

    1. **Modify login()** — Add MFA branch:
       - After successful password verification and email verification check, BEFORE token issuance:
       - Check if user.mfa_enabled is True
       - If MFA enabled:
         a. Check for trusted device cookie: call `is_device_trusted(redis, user.id, device_token)` where device_token comes from a new optional parameter `device_token: str | None = None`
         b. If device is trusted AND not in an enforcing tenant context (tenant enforcement is checked in Plan 04; for now, just check device trust), skip MFA and proceed to token issuance
         c. If device NOT trusted: generate an opaque mfa_pending token (`secrets.token_urlsafe(32)`), store in Redis at `auth:mfa_pending:{token}` with value `str(user.id)` and TTL = `settings.MFA_PENDING_TTL_SECONDS` (300s)
         d. Return a dict/object with `mfa_required=True, mfa_token=token` (no access/refresh tokens)
       - If MFA not enabled: proceed with normal token issuance as before
       - Change the return type to support both cases. Return a dict that the router interprets.

       IMPORTANT: The login() function signature needs to accept an optional device_token parameter. The router will extract this from the cookie.

    2. **mfa_verify(db, redis, mfa_token, code, trust_device) -> dict** — New function:
       - Look up `auth:mfa_pending:{mfa_token}` in Redis
       - If not found, raise InvalidTokenError() (expired or invalid)
       - Extract user_id from Redis value
       - Load User from DB; if not found or not active, raise InvalidTokenError()
       - **TOTP replay check**: Check Redis key `auth:mfa:used:{user_id}`. If it exists AND its value equals `code`, raise MfaInvalidCodeError() (replay detected). Only check for 6-digit TOTP codes (not backup codes).
       - **Verify code** — try TOTP first, then backup code:
         a. TOTP: `pyotp.TOTP(user.mfa_secret).verify(code, valid_window=1)`. If True, store replay prevention key: `auth:mfa:used:{user_id}` = code with TTL 60s.
         b. Backup code: if TOTP fails, strip dashes from code, query MfaBackupCode where user_id matches and used_at is NULL, iterate rows and `verify_password(code_stripped, row.code_hash)`. If match, set `row.used_at = datetime.now(tz=timezone.utc)`.
         c. If neither matches, raise MfaInvalidCodeError()
       - Delete the `auth:mfa_pending:{mfa_token}` Redis key (consumed)
       - Issue tokens via `_issue_tokens(db, redis, user)`
       - If trust_device is True, call `create_trusted_device(db, user.id)` to generate device token
       - Return dict with tokens + optional device_token for cookie setting

    3. **Trusted device helpers**:

       **create_trusted_device(db, user_id) -> str** — Create trusted device record:
       - Generate device_token: `secrets.token_urlsafe(32)`
       - Hash it: `hashlib.sha256(device_token.encode()).hexdigest()`
       - Create TrustedDevice row: user_id, token_hash, expires_at = now + timedelta(days=settings.TRUSTED_DEVICE_TTL_DAYS)
       - Return the plaintext device_token (for cookie)

       **is_device_trusted(db, user_id, device_token) -> bool** — Check if device is trusted:
       - If device_token is None or empty, return False
       - Hash the token: `hashlib.sha256(device_token.encode()).hexdigest()`
       - Query TrustedDevice by token_hash where user_id matches
       - If not found, return False
       - Check expires_at: if expired, delete the row and return False
       - If valid, return True
       - Note: This is an async function since it queries the DB

       **revoke_trusted_devices(db, user_id) -> None** — Delete all trusted devices for a user:
       - `await db.execute(delete(TrustedDevice).where(TrustedDevice.user_id == user_id))`

       Import TrustedDevice from auth/models.py.

       - Audit: only log successful login (when tokens are issued, not when mfa_required is returned). When mfa_required, log action="login_mfa_pending".

       **Add POST /auth/mfa/verify** — MFA verification endpoint:
       - Accept `MfaVerifyRequest` body (mfa_token, code, trust_device)
       - Call `service.mfa_verify(db, redis, body.mfa_token, body.code, body.trust_device)`
       - Returns a JSONResponse (not Pydantic model) to allow setting cookies
       - Build the response JSON with access_token, refresh_token, token_type="bearer"
       - If trust_device was True and result contains `device_token`:
         ```python
         response.set_cookie(

             secure=settings.APP_ENV != "development",  # False in dev, True in production
             samesite="lax",
             max_age=settings.TRUSTED_DEVICE_TTL_DAYS * 86400,
         )
         ```
       - Rate limit: `@limiter.limit(settings.RATE_LIMIT_AUTH)` (prevent brute-force on 6-digit TOTP)
