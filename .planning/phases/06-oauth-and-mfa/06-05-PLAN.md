       - `test_mfa_status`: GET /auth/mfa/status before and after enrollment. Assert correct mfa_enabled value.

       - `test_mfa_verify_with_backup_code`: Use mfa_token + a backup code. Assert tokens returned. Assert backup code marked as used.
       - `test_mfa_verify_replay_rejected`: Verify with a TOTP code, then immediately try the same code again. Assert second attempt fails with MFA_INVALID_CODE.
       - `test_mfa_verify_expired_token_fails`: Set a very short TTL or wait, or directly delete the Redis key. Assert 401.

       - `test_owner_with_mfa_enables_enforcement`: Owner with MFA enabled calls the endpoint. Assert tenant.mfa_enforced=True.
       - `test_enforcement_revokes_non_mfa_sessions`: Create tenant with 2 members (owner has MFA, member does not). Enable enforcement. Assert member's refresh token is deleted.
       - `test_enforcement_login_without_mfa_signals_setup_required`: User in enforcing tenant without MFA logs in. Assert mfa_setup_required=True.
