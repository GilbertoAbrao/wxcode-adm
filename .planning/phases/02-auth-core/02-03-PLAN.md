3. **auth/service.py** — Add login, refresh, logout functions and blacklist logic:

   **Access token blacklist helpers:**
   - `async def blacklist_access_token(redis: Redis, token: str) -> None`:
     - Decode token to get jti and exp (use `jwt.decode` with `options={"verify_exp": False}` since we need to read exp even if expired)
     - Actually: use `decode_access_token` but catch TokenExpiredError and still extract jti — OR directly use `jwt.decode` with verify_exp=False here
     - Better approach: `payload = jwt.decode(token, settings.JWT_PUBLIC_KEY.get_secret_value(), algorithms=["RS256"], options={"verify_exp": False})`
     - Calculate remaining TTL: `remaining = int(payload["exp"] - datetime.now(timezone.utc).timestamp())`
     - If remaining > 0: `await redis.set(f"auth:blacklist:jti:{payload['jti']}", "1", ex=remaining)`

   - `async def is_token_blacklisted(redis: Redis, jti: str) -> bool`:
     - `return await redis.exists(f"auth:blacklist:jti:{jti}") > 0`

   **Login function:**
   - `async def login(db: AsyncSession, redis: Redis, body: LoginRequest) -> TokenResponse`:
     - Find user by email — if not found, raise `InvalidCredentialsError()`
     - If not user.is_active, raise `InvalidCredentialsError()` (same message)
     - Verify password: `verify_password(body.password, user.password_hash)` — if False, raise `InvalidCredentialsError()`
     - If not user.email_verified, raise `EmailNotVerifiedError(error_code="EMAIL_NOT_VERIFIED", message="Please verify your email before logging in")`
     - **Single-session enforcement:** Delete ALL existing refresh tokens for this user:
       `await db.execute(delete(RefreshToken).where(RefreshToken.user_id == user.id))`
     - Create access token: `access_token = create_access_token(str(user.id))`
     - Create refresh token: `refresh_token_str = secrets.token_urlsafe(32)`
     - Store refresh token: `db.add(RefreshToken(token=refresh_token_str, user_id=user.id, expires_at=datetime.now(timezone.utc) + timedelta(days=settings.REFRESH_TOKEN_TTL_DAYS)))`
     - Return `TokenResponse` schema values (access_token, refresh_token_str)
     - Note: `db.flush()` is not strictly needed here because `get_session` commits after the endpoint returns

   **Refresh function:**
   - `async def refresh(db: AsyncSession, redis: Redis, body: RefreshRequest) -> TokenResponse`:
     - Find refresh token row: `select(RefreshToken).where(RefreshToken.token == body.refresh_token)`
     - If not found → REPLAY DETECTED:
       - We need the user_id to revoke all sessions. Problem: if the token was deleted, we don't know the user.
       - Solution: Decode the old access token from the request header? No — refresh doesn't require access token.
       - Better: The missing token could be a random string (not a real replay). Just raise `InvalidTokenError()`.
       - Actually per research: "on replay, delete ALL user tokens for that user". But if the token is not found, we don't know the user. The practical approach is: if not found, raise `InvalidTokenError()`. True replay detection works because: attacker replays consumed token -> not found -> error. The legitimate user already has the new token. If the attacker somehow gets the user_id, they still can't revoke because they don't have a valid token.
       - Raise `InvalidTokenError()`
     - If found but expired (`row.expires_at < datetime.now(timezone.utc)`): delete the row, raise `TokenExpiredError()`
     - **Rotation:** Delete the old token row, create a new one:
       - `await db.delete(row)`
       - `new_token_str = secrets.token_urlsafe(32)`
       - `db.add(RefreshToken(token=new_token_str, user_id=row.user_id, expires_at=datetime.now(timezone.utc) + timedelta(days=settings.REFRESH_TOKEN_TTL_DAYS)))`
     - Create new access token: `access_token = create_access_token(str(row.user_id))`
     - Return TokenResponse values

   **Logout function:**
   - `async def logout(db: AsyncSession, redis: Redis, refresh_token: str, access_token: str) -> None`:
     - Find and delete the refresh token row: `select(RefreshToken).where(RefreshToken.token == refresh_token)` — delete if found (ignore if not found, idempotent)
     - Blacklist the access token: `await blacklist_access_token(redis, access_token)`
