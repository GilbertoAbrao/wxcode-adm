1. **auth/dependencies.py** — Create FastAPI auth dependencies:

   ```python
   import uuid
   from fastapi import Depends, HTTPException, status
   from fastapi.security import OAuth2PasswordBearer
   from redis.asyncio import Redis
   from sqlalchemy.ext.asyncio import AsyncSession

   from wxcode_adm.auth.exceptions import TokenExpiredError, InvalidTokenError, EmailNotVerifiedError
   from wxcode_adm.auth.jwt import decode_access_token
   from wxcode_adm.auth.models import User
   from wxcode_adm.auth.service import is_token_blacklisted
   from wxcode_adm.dependencies import get_session, get_redis
   ```

   - `oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/api/v1/auth/login", auto_error=True)`

   - `async def get_current_user(token: str = Depends(oauth2_scheme), db: AsyncSession = Depends(get_session), redis: Redis = Depends(get_redis)) -> User`:
     - Decode token: `payload = decode_access_token(token)` — raises TokenExpiredError or InvalidTokenError (caught by AppError handler)
     - Extract `sub` (user_id) and `jti` — if missing, raise `InvalidTokenError()`
     - Check blacklist: `if await is_token_blacklisted(redis, jti): raise InvalidTokenError()`
     - Load user: `user = await db.get(User, uuid.UUID(sub))`
     - If user is None or not user.is_active: raise `InvalidTokenError()`
     - Return user

   - `async def require_verified(user: User = Depends(get_current_user)) -> User`:
     - If not user.email_verified: raise `EmailNotVerifiedError(error_code="EMAIL_NOT_VERIFIED", message="Email verification required")`
     - Return user

2. **auth/router.py** — Add a protected test endpoint (temporary, useful for integration testing):
   - `@auth_api_router.get("/me")`:
     - Params: `user: User = Depends(require_verified)`
     - Return `{"id": str(user.id), "email": user.email, "email_verified": user.email_verified}`
   - This endpoint proves the full dependency chain works. It will be refined in Phase 7 (User Account).

3. **Alembic migration** — Generate the migration for users and refresh_tokens tables:
   - First ensure auth models are imported in alembic env.py. Check if `from wxcode_adm.auth.models import User, RefreshToken` is needed.
   - Read the existing alembic/env.py to understand the import pattern.
   - Run: `cd backend && alembic revision --autogenerate -m "add users and refresh_tokens tables"`
   - Verify the generated migration has CreateTable for both `users` and `refresh_tokens`.
   - Do NOT run `alembic upgrade head` — that requires a live database. The migration file existence is sufficient for this plan.

4. **main.py** — No changes needed if auth_api_router was already registered in Plan 02. Verify the JWKS router (root level) and auth_api_router (API prefix) are both present.

     - Signup + verify, POST /api/v1/auth/login with wrong password

     - Login, POST /api/v1/auth/refresh with refresh_token

   - `test_refresh_rejects_consumed_token`:

   **Test SC5: Logout and blacklist**

     - Signup + verify + login, forgot-password, extract reset token from Redis/service call
